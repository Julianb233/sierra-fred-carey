## Tech Debt (Non-Blocking)

### P2 — SMS Nav Tier Mismatch (Cosmetic)

**File:** `app/dashboard/layout.tsx` lines 112-118
**Issue:** "Weekly Check-ins" nav item shows `tier: 1` (Pro) and `badge: "Pro"`, but SMS requires Studio tier. API and FeatureLock correctly enforce Studio.
**Impact:** Pro users can navigate to the page but see a lock overlay. Misleading badge.
**Fix:** Change `tier: 1` → `tier: 2`, badge `"Pro"` → `"Studio"`.

### P2 — Password Validation Mismatch (UX)

**Files:** `app/get-started/page.tsx` (6+ chars), `app/api/onboard/route.ts` (8+ chars + uppercase + digit)
**Impact:** Users entering 6-7 char lowercase passwords pass client validation but fail server.
**Fix:** Align client to match server requirements.

### P3 — Onboarding Session Continuity (Runtime Risk)

**File:** `app/get-started/page.tsx`
**Issue:** After server-side signUp, redirect to /dashboard may not have session cookie set if Supabase SSR isn't configured correctly. `middleware.ts` is deleted.
**Impact:** Requires runtime testing to confirm. Likely works via Supabase SSR cookie management.

### P3 — Missing VERIFICATION.md (Documentation)

**Phases:** 01, 02 — executed before verification framework. Artifacts confirmed substantive.
