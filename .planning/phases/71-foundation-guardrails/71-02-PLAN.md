## Steps

### Step 1: Refactor prompt into immutable core + mutable supplemental layer

**Files:**
- `lib/ai/prompts.ts` (modify)
- `lib/ai/prompt-layers.ts` (new)

The current `FRED_CAREY_SYSTEM_PROMPT` in `lib/ai/prompts.ts` is a single large template literal. Refactor into two layers:

**`lib/ai/prompt-layers.ts`:**
```typescript
/**
 * FRED Core Prompt — IMMUTABLE
 *
 * This constant defines FRED's identity, voice, and coaching methodology.
 * It MUST NOT be modified by any optimization, A/B test, or feedback loop.
 * Changes to this constant require manual review and voice regression suite passing.
 *
 * To add supplemental guidance, use the SUPPLEMENTAL_PROMPT_LAYER below.
 */
export const FRED_CORE_PROMPT = Object.freeze({
  // Move the entire current FRED_CAREY_SYSTEM_PROMPT content here
  // Marked with Object.freeze to signal immutability at runtime
  content: `...existing prompt content...`,
  version: '1.0.0',
  lastModified: '2026-03-06',
  frozen: true as const,
})

/**
 * Supplemental Prompt Layer — MUTABLE
 *
 * This layer is for feedback-driven prompt patches, A/B test variants,
 * and topic-specific additions. It is appended AFTER the core prompt.
 *
 * Each entry has:
 * - id: unique identifier
 * - content: the supplemental instruction text
 * - source: what created this patch (feedback cluster ID, A/B experiment, manual)
 * - active: whether to include in prompt assembly
 * - createdAt: when the patch was created
 */
export interface SupplementalPromptPatch {
  id: string
  content: string
  source: 'feedback' | 'ab_test' | 'manual'
  sourceId?: string  // feedback_insight.id or ab_experiment.id
  active: boolean
  createdAt: string
}

// Start empty — patches added via admin dashboard in Phase 76
export const SUPPLEMENTAL_PATCHES: SupplementalPromptPatch[] = []
```

**Modify `lib/ai/prompts.ts`:**
- Import `FRED_CORE_PROMPT` from `./prompt-layers`
- Replace `FRED_CAREY_SYSTEM_PROMPT` with a getter that returns `FRED_CORE_PROMPT.content`
- Keep the existing `export const FRED_CAREY_SYSTEM_PROMPT` as an alias pointing to `FRED_CORE_PROMPT.content` so all existing imports continue to work (zero breaking changes)
- Update `buildSystemPrompt()` to assemble: core prompt + active supplemental patches + founder context
- Add `buildPromptWithSupplements(founderContext: string, patches: SupplementalPromptPatch[]): string` function

**Critical constraint:** Do NOT change the actual prompt text. This is a structural refactor only. The existing `prompts.test.ts` tests (all 314 lines) must continue to pass unchanged.

### Step 2: Build voice regression test suite (20+ scenarios)

**Files:** `lib/ai/__tests__/voice-regression.test.ts` (new)

Create a comprehensive test suite that validates FRED's voice characteristics. These tests assert on the prompt content (not LLM output — that would be flaky and expensive).

The suite must cover these FRED voice dimensions with 20+ individual test cases:

**Group 1: Blunt Truth-Telling (5+ tests)**
- Prompt contains "direct, occasionally blunt" or equivalent
- Prompt contains instruction to not sugarcoat
- Prompt contains "Evidence > Narrative" principle
- Prompt does NOT contain "always be positive" or toxic positivity language
- Prompt contains "challenge assumptions" instruction

**Group 2: Reframe-Before-Prescribe (5+ tests)**
- Prompt contains "Reframe before prescribe" principle
- Prompt contains "Never answer the surface question" instruction
- Prompt contains diagnostic questioning methodology
- Prompt contains instruction to understand before advising
- Prompt contains "ask why" or equivalent Socratic method reference

**Group 3: Mentor Tone (5+ tests)**
- Prompt contains "mentor" role reference
- Prompt contains "Encourage without flattery" principle
- Prompt contains "tough love" or "care enough to be honest" sentiment
- Prompt does NOT contain "always agree" or sycophantic instructions
- Prompt contains experience-based credibility references (years, companies)

**Group 4: Coaching Boundaries (5+ tests)**
- Prompt contains "Do not encourage fundraising by default" (capital-as-tool principle)
- Prompt contains "Never optimize downstream artifacts" (decision sequencing)
- Prompt contains startup process step awareness
- Prompt contains tier-appropriate coaching depth references
- Prompt contains wellbeing/red-flag detection references

**Group 5: Immutability Verification (3+ tests)**
- `FRED_CORE_PROMPT.frozen` is `true`
- `FRED_CORE_PROMPT.version` matches expected version
- `FRED_CORE_PROMPT.content` is identical to a hash/snapshot (store SHA-256 of the prompt as a constant in the test file — if anyone modifies the core prompt, this test fails and forces review)
- `Object.isFrozen(FRED_CORE_PROMPT)` returns `true`

Each test must have a descriptive name that explains what voice characteristic it protects. Example: `it('rejects sugarcoating — FRED never uses toxic positivity language')`.
