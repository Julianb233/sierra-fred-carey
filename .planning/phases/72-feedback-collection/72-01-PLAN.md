**hooks/use-feedback.ts** — Client hook managing feedback state and submission.

Props/args: `{ messageId: string, userId?: string }`

State:
- `rating`: `null | 'up' | 'down'` — current thumbs selection
- `expanded`: `boolean` — whether expansion panel is visible
- `category`: `string | null` — selected category (thumbs-down only)
- `comment`: `string` — free-text (both up and down)
- `submitted`: `boolean` — locks widget after submit
- `canShowWidget`: `boolean` — derived from message count gate

Functions:
- `setThumbsUp()`: Sets rating='up', expands panel for optional comment. If user submits empty comment, sends signal immediately (fire-and-forget).
- `setThumbsDown()`: Sets rating='down', expands panel for category + comment.
- `submitFeedback()`: POSTs to `/api/feedback/signal` via fire-and-forget fetch (no await in the UI — use `.catch(console.error)` pattern like `logJourneyEventAsync`). Sets submitted=true immediately (optimistic). Body: `{ message_id, signal_type, rating, category, comment }`.
- `dismiss()`: Collapses panel without submitting detailed feedback. If rating was already set, the thumbs signal is already sent.

Message count gate: Accept a `messageCount` prop. If `messageCount < MIN_MESSAGES_FOR_FEEDBACK` (5, from constants), `canShowWidget` is false. Import `MIN_MESSAGES_FOR_FEEDBACK` from `@/lib/feedback/constants`.

Key design decision: When user clicks thumbs-up or thumbs-down, IMMEDIATELY fire-and-forget the basic signal (rating only). The expansion panel is for OPTIONAL additional detail. This way even a single tap produces a signal. Do NOT wait for the expansion form.

**components/feedback/thumbs-widget.tsx** — Inline widget rendered below assistant messages.

Props: `{ messageId: string, messageCount: number }`

Renders:
- Two icon buttons: ThumbsUp and ThumbsDown from lucide-react
- Icons are 14px (`h-3.5 w-3.5`), muted by default (`text-muted-foreground/40`), hover to `text-muted-foreground`
- Active state: thumbs-up turns `text-green-500`, thumbs-down turns `text-red-400`
- Use `cn()` from `@/lib/utils` for conditional classes
- Wrap in `motion.div` with `initial={{ opacity: 0 }} animate={{ opacity: 1 }}` for subtle fade-in
- When `canShowWidget` is false (from useFeedback), render null
- When `submitted` is true, show a small checkmark icon or "Thanks!" text (text-xs, muted) then fade out after 2 seconds
- Below the thumbs row, conditionally render `<FeedbackExpansion>` when `expanded` is true
- Use `AnimatePresence` + `motion.div` for smooth expand/collapse animation (height auto via layout animation)

**components/feedback/feedback-expansion.tsx** — Expandable detail panel.

Props: `{ variant: 'up' | 'down', category: string | null, comment: string, onCategoryChange: (cat: string) => void, onCommentChange: (text: string) => void, onSubmit: () => void, onDismiss: () => void }`

Renders:
- For variant='down': Row of pill buttons for categories: irrelevant, incorrect, too vague, too long, wrong tone. Use `FEEDBACK_CATEGORIES` from `@/lib/feedback/constants` but filter to the negative ones (exclude 'helpful', 'other', 'coaching_discomfort'). Each pill: `text-xs px-2 py-1 rounded-full border`. Selected pill: `bg-[#ff6a1a]/10 border-[#ff6a1a]/40 text-[#ff6a1a]`. Unselected: `bg-white/5 border-white/20 text-muted-foreground`.
- For variant='up': No category selector.
- Both: Textarea for comment. Placeholder: "What was helpful?" (up) or "Tell us more (optional)" (down). Max height 80px. `text-xs`. Use a plain `<textarea>` with Tailwind, not a heavy component.
- Submit button: "Send" text-xs, small, Sahara orange background (`bg-[#ff6a1a]`), white text. Only shown after user has interacted (category selected or comment typed). For thumbs-up, show after any text is typed.
- Dismiss X button (top right) — calls onDismiss.
- Entire panel: `rounded-lg border border-white/10 bg-white/5 backdrop-blur-sm p-3 mt-2`

Performance: Use `React.lazy` if needed, but given the small size, static import is fine. Keep the components lean — no external form libraries.
