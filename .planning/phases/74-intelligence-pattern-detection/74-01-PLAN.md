1. `clusterFeedbackSignals(since: string): Promise<ClusteringResult>`:
   - Main orchestrator. Steps:
     a. Call `getRecentNegativeSignals(since)` -- if empty, return early with zero counts
     b. Call `groupSignalsByTheme(signals)` to get `FeedbackCluster[]`
     c. Fetch open insights once via `getOpenInsights()` for dedup checks
     d. For each cluster, call `findDuplicateInsight(cluster, openInsights)`
     e. If duplicate found: merge signal_ids (union of existing + new, deduplicated via Set), call `updateInsightSignals()` with merged array and new count, increment `clustersDeduped`
     f. If no duplicate: compute severity via `computeClusterSeverity()`, then call `insertFeedbackInsight()` with:
        - `insight_type: 'cluster'`
        - `title`: cluster.theme
        - `description`: cluster.description
        - `category`: cluster.category
        - `severity`: computed severity
        - `signal_count`: cluster.signalIds.length
        - `signal_ids`: cluster.signalIds
        - `status: 'new'`
        - `linear_issue_id: null`
        - `actioned_at: null`
        - `resolved_at: null`
        - `metadata: { source: 'daily-clustering', run_at: new Date().toISOString() }`
     g. Wrap each cluster processing in try/catch -- log errors but continue processing remaining clusters
     h. Return `ClusteringResult` with counts

2. `groupSignalsByTheme(signals: FeedbackSignal[]): Promise<FeedbackCluster[]>`:
   - If `signals.length < 3`, create a single cluster with all signals titled "Uncategorized feedback" (too few to meaningfully cluster)
   - For 3+ signals: call Anthropic API using the same fetch pattern as `trigger/sahara-whatsapp-monitor.ts` `identifyIssues()`:
     ```typescript
     const apiKey = process.env.ANTHROPIC_API_KEY;
     if (!apiKey) {
       console.warn("[Clustering] ANTHROPIC_API_KEY not set, using category fallback");
       return groupByCategoryFallback(signals);
     }

     const response = await fetch("https://api.anthropic.com/v1/messages", {
       method: "POST",
       headers: {
         "Content-Type": "application/json",
         "x-api-key": apiKey,
         "anthropic-version": "2023-06-01",
       },
       body: JSON.stringify({
         model: "claude-sonnet-4-20250514",
         max_tokens: 4096,

       }),
     });
     ```
   - Prompt: Build a summary of each signal (id, category, comment or "no comment", user_tier, created_at date). Ask the LLM:
     ```
     You are analyzing negative feedback signals from the Sahara AI coaching platform (joinsahara.com).
     Group these signals by common theme/issue. For each group, provide:
     - theme: A concise title (5-10 words) describing the issue
     - description: 1-2 sentence summary of what users are complaining about
     - category: The most common feedback category in this group (one of: irrelevant, incorrect, too_vague, too_long, wrong_tone, coaching_discomfort, other) or null if mixed
     - signal_ids: Array of signal IDs that belong to this group

     Every signal must appear in exactly one group. Create 1-5 groups maximum.
     If all signals share the same complaint, use 1 group.
     Return ONLY a JSON array of objects.
     ```
   - Parse JSON from response (same `content.match(/\[[\s\S]*\]/)` pattern as WhatsApp monitor)
   - Map signal_ids back to actual FeedbackSignal objects from the input array
   - Validate that returned signal_ids exist in the input set; discard any that don't match
   - **Fallback** (`groupByCategoryFallback`): If LLM call fails or response cannot be parsed, group by `category` field:
     - Group signals by their `category` value (null becomes "uncategorized")
     - Use the category name as theme, generate description like "N feedback signals categorized as {category}"
     - Return FeedbackCluster[] from the grouping

3. `computeClusterSeverity(cluster: FeedbackCluster): 'low' | 'medium' | 'high' | 'critical'`:
   ```typescript
   export function computeClusterSeverity(
     cluster: FeedbackCluster
   ): 'low' | 'medium' | 'high' | 'critical' {
     const weightedImpact = cluster.signals.reduce(
       (sum, s) => sum + (TIER_WEIGHTS[s.user_tier as keyof typeof TIER_WEIGHTS] ?? 1),
       0
     );
     const count = cluster.signals.length;
     if (weightedImpact >= 15 || count >= 10) return 'critical';
     if (weightedImpact >= 8 || count >= 5) return 'high';
     if (weightedImpact >= 3 || count >= 3) return 'medium';
     return 'low';
   }
   ```

4. `findDuplicateInsight(cluster: FeedbackCluster, openInsights: FeedbackInsight[]): FeedbackInsight | null`:
   - Two-pass dedup:
     a. **Category + time window**: If cluster.category is non-null and matches an insight's category, AND the insight was created within the last 4 hours --> return that insight as duplicate
     b. **Title word-overlap**: Tokenize cluster.theme and each insight.title to lowercase words (split on `/\W+/`), compute Jaccard similarity = `|intersection| / |union|`. If similarity > 0.5 AND insight was created within the last 24 hours --> return that insight
   - Helper function `jaccardSimilarity(a: string, b: string): number`:
     ```typescript
     function jaccardSimilarity(a: string, b: string): number {
       const setA = new Set(a.toLowerCase().split(/\W+/).filter(Boolean));
       const setB = new Set(b.toLowerCase().split(/\W+/).filter(Boolean));
       const intersection = new Set([...setA].filter(x => setB.has(x)));
       const union = new Set([...setA, ...setB]);
       return union.size > 0 ? intersection.size / union.size : 0;
     }
     ```
   - Return the first matching insight or null
   - NOTE: Simple word-overlap rather than embeddings. For Sahara's volume (<500 signals/day) this is sufficient. Embedding-based dedup can be added in v8.

```typescript
/**
 * Scheduled task: Feedback Clustering & Pattern Detection
 *
 * Runs daily at 6 AM UTC to:
 * 1. Fetch negative feedback signals from the past 24 hours
 * 2. Use AI to cluster signals by theme
 * 3. Deduplicate against existing open insights
 * 4. Write new clusters to feedback_insights table
 *
 * Feeds into: admin "Top Issues This Week" dashboard, Linear auto-triage (74-02)
 */
import { schedules, logger } from "@trigger.dev/sdk/v3";

export const feedbackClusteringJob = schedules.task({
  id: "feedback-clustering-daily",
  // Daily at 6 AM UTC (11 PM PT previous day)
  cron: "0 6 * * *",
  maxDuration: 300, // 5 minutes
  retry: {
    maxAttempts: 2,
    minTimeoutInMs: 5000,
    maxTimeoutInMs: 30000,
    factor: 2,
  },
  run: async (payload) => {
    logger.log("Feedback clustering job started", {
      timestamp: payload.timestamp,
    });

    try {
      // Dynamic import to avoid bundling issues in Trigger.dev
      const { clusterFeedbackSignals } = await import("@/lib/feedback/clustering");

      // Process signals from the last 24 hours
      const since = new Date();
      since.setHours(since.getHours() - 24);

      const result = await clusterFeedbackSignals(since.toISOString());

      logger.log("Feedback clustering job completed", {
        clustersCreated: result.clustersCreated,
        clustersDeduped: result.clustersDeduped,
        signalsProcessed: result.signalsProcessed,
        errors: result.errors,
      });

      if (result.errors.length > 0) {
        logger.warn("Clustering completed with errors", {
          errorCount: result.errors.length,
          errors: result.errors,
        });
      }

      return result;
    } catch (err) {
      const msg = `Feedback clustering job failed: ${err}`;
      logger.error(msg);
      throw err; // Let Trigger.dev handle retries
    }
  },
});
```

Key design decisions:
- 24-hour lookback window -- process yesterday's feedback each morning
- 5-minute max duration (less than WhatsApp monitor's 10m since no browser automation)
- 2 retry attempts with exponential backoff
- Dynamic import of clustering.ts to avoid bundling issues in Trigger.dev runtime
- Fire at 6 AM UTC so results are ready for admin's morning review
- Errors are logged but also re-thrown so Trigger.dev retry logic kicks in
