**Add fire-and-forget memory extraction to the chat route after each completed response.**

The chat route has two code paths: non-streaming and streaming. Both need the same post-response extraction.

1. Import at top of route file:
   ```typescript
   import { extractMemoryUpdates, persistExtractedFacts } from "@/lib/fred/active-memory";
   ```

2. Create a helper function in the route file (after the existing `fireEnrichment` helper):

   ```typescript
   /**
    * Fire-and-forget: Extract structured facts from the chat exchange using LLM
    * and persist to semantic memory. Only runs for Pro+ tiers with persistent memory.
    * This complements the heuristic extractProfileEnrichment (regex-based) with
    * LLM-powered extraction that catches facts regex misses.
    */
   function fireMemoryExtraction(
     userId: string,
     userMessage: string,
     assistantMessage: string
   ): void {
     (async () => {
       try {
         const facts = await extractMemoryUpdates(userId, userMessage, assistantMessage);
         const factCount = Object.keys(facts).length;
         if (factCount > 0) {
           const stored = await persistExtractedFacts(userId, facts);
           if (stored > 0) {
             console.log(`[Active Memory] Extracted and stored ${stored} facts for user ${userId.slice(0, 8)}...`);
           }
         }
       } catch (error) {
         console.warn("[Active Memory] Extraction pipeline failed (non-blocking):", error);
       }
     })();
   }
   ```

3. **Non-streaming path** (around line 760, after the existing `fireEnrichment` call):
   Add after the `fireEnrichment(userId, [...])` call:

   ```typescript
   // Phase 79: Fire-and-forget LLM-based memory extraction (Pro+ only)
   if (shouldPersistMemory) {
     fireMemoryExtraction(userId, message, result.response.content);
   }
   ```

4. **Streaming path** (around line 1020, after the `storeEpisode` for assistant response, inside the `if (update.isComplete)` block):
   Add after the existing assistant memory store:

   ```typescript
   // Phase 79: Fire-and-forget LLM-based memory extraction (Pro+ only)
   if (shouldPersistMemory) {
     fireMemoryExtraction(userId, message, response.content);
   }
   ```

5. **DO NOT** modify the system prompt assembly section. That's handled by Task 1 via the context-builder changes which flow through `buildFounderContextWithFacts` -> `founderContext` -> `fullContext`.

6. **DO NOT** remove the existing `fireEnrichment` call. The heuristic extractor is fast (no API call) and catches structured data like "$50k MRR" that the LLM might miss or format differently. Both extractors complement each other:
   - `fireEnrichment` -> regex extraction -> profiles.enrichment_data (fast, free)
   - `fireMemoryExtraction` -> LLM extraction -> fred_semantic_memory (slower, costs tokens, richer)
