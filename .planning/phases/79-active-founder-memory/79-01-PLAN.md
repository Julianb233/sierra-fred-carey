**2. Create `lib/fred/active-memory.ts`:**

This module builds a FounderContext by merging data from three sources in priority order:
1. `profiles` table (onboarding data -- highest priority for fields it has)
2. `fred_semantic_memory` table (chat-extracted facts)
3. `enrichment_data` JSONB on profiles (heuristic extraction)

Key function: `buildActiveFounderContext(userId: string): Promise<FounderContext>`

Steps inside buildActiveFounderContext:
- Load profile from `profiles` table (same query as existing `loadFounderProfile` in context-builder.ts, but also SELECT `co_founder`)
- Load all semantic facts via `getAllUserFacts(userId)`
- For each FounderContext field, resolve value from best available source
- Compute confidence for each field using `computeConfidence(lastUpdated: Date | null): MemoryConfidence`:
  - No value -> "unknown"
  - Updated < 7 days ago -> "high"
  - Updated 7-30 days ago -> "medium"
  - Updated > 30 days ago -> "low"

Helper: `computeConfidence(lastUpdated: Date | null): MemoryConfidence` -- exported for testing.

Helper: `formatFounderContextBlock(ctx: FounderContext): string` -- produces the text block that replaces `{{FOUNDER_CONTEXT}}` in the system prompt. Format:

```
## ACTIVE FOUNDER CONTEXT

**Founder:** {name} | **Company:** {company} | **Stage:** {stage}
**Market:** {market} | **Co-Founder:** {coFounder} | **Team:** {teamSize}
**Biggest Challenge:** {challenge}
**90-Day Goal:** {goal}
**Product:** {productStatus} | **Revenue:** {revenueStatus} | **Funding:** {fundingStatus}

### Stale Context (needs re-confirmation)
{For each field with confidence "low" or "medium", list it here with instruction:}
- "{fieldName}" was last updated {N} days ago. Ask: "Last time we talked, you mentioned {value}. Is that still the case?"

### Missing Context (gather naturally)
{For each field with confidence "unknown", list it here:}
- "{fieldName}" is unknown. Weave this into the next exchange naturally.

CRITICAL INSTRUCTION: You MUST reference the founder's specific context above in EVERY response. Use their company name, their market, their stage, their challenge. No generic answers. If you catch yourself giving advice that could apply to any founder, stop and re-anchor to their specific situation.
```

For fields with "unknown" confidence, omit them from the top section and list them in "Missing Context".
For fields with "low" confidence, show them in top section AND in "Stale Context" with re-confirmation prompt.

**1. Create `lib/ai/memory-extraction-prompt.ts`:**

This module provides the system prompt used to extract structured facts from a chat message pair (user message + FRED response). It uses a SMALL, FAST model (same as Free tier -- GPT-4o-mini or equivalent) to keep costs low since this runs on every chat turn.

```typescript
export const MEMORY_EXTRACTION_PROMPT = `You are a fact extraction engine. Given a conversation exchange between a founder and their AI mentor, extract any NEW or UPDATED facts about the founder and their startup.

Extract ONLY facts explicitly stated or strongly implied. Do not infer or guess.

Return a JSON object with ONLY the fields that have new/updated information. Omit fields where no new info was found.

Fields to extract:
- companyName: The name of the startup/company
- stage: Startup stage (idea, pre-seed, seed, series-a, growth, etc.)
- market: Industry or market sector
- coFounder: Co-founder name(s), or "solo founder" if explicitly stated
- biggestChallenge: Their current biggest challenge or blocker
- ninetyDayGoal: What they want to accomplish in the next 90 days
- productStatus: Current product state (idea, MVP, beta, launched, scaling)
- revenueStatus: Revenue information (pre-revenue, $X MRR, etc.)
- teamSize: Number of team members
- fundingStatus: Funding state (bootstrapped, raised $X, seeking seed, etc.)

IMPORTANT RULES:
- Only extract what is EXPLICITLY stated or clearly implied in THIS exchange
- If the founder corrects a previous fact, extract the CORRECTED value
- If no new facts are found, return an empty JSON object: {}
- Return ONLY valid JSON, no markdown, no explanation

Example input:
User: "We just closed our seed round at $2M. Team is now 8 people. The biggest thing keeping me up at night is our churn rate."
Assistant: "Congratulations on closing the round..."

Example output:
{"fundingStatus":"raised $2M seed round","teamSize":"8","biggestChallenge":"high churn rate"}`;

export function parseExtractionResult(raw: string): ExtractedFacts {
  try {
    // Strip markdown code blocks if present
    const cleaned = raw.replace(/```json?\n?/g, "").replace(/```/g, "").trim();
    const parsed = JSON.parse(cleaned);
    // Validate: only keep known fields with string values
    const VALID_KEYS = ["companyName", "stage", "market", "coFounder", "biggestChallenge", "ninetyDayGoal", "productStatus", "revenueStatus", "teamSize", "fundingStatus"];
    const result: Record<string, string> = {};
    for (const key of VALID_KEYS) {
      if (typeof parsed[key] === "string" && parsed[key].trim().length > 0) {
        result[key] = parsed[key].trim();
      }
    }
    return result as ExtractedFacts;
  } catch {
    return {};
  }
}
```

```typescript
export async function extractMemoryUpdates(
  userId: string,
  userMessage: string,
  assistantMessage: string
): Promise<ExtractedFacts> {
  // Use a small fast model (same as free tier) for cost efficiency
  const { generateText } = await import("ai");
  const { openai } = await import("@ai-sdk/openai");
  const { MEMORY_EXTRACTION_PROMPT, parseExtractionResult } = await import("@/lib/ai/memory-extraction-prompt");

  try {
    const result = await generateText({
      model: openai("gpt-4o-mini"),
      system: MEMORY_EXTRACTION_PROMPT,
      prompt: `User: "${userMessage}"\nAssistant: "${assistantMessage}"`,
      maxTokens: 500,
      temperature: 0,
    });
    return parseExtractionResult(result.text);
  } catch (error) {
    console.warn("[Active Memory] Extraction failed (non-blocking):", error);
    return {};
  }
}
```

```typescript
import { storeFact, type SemanticCategory } from "@/lib/db/fred-memory";

const FACT_TO_SEMANTIC: Record<keyof ExtractedFacts, { category: SemanticCategory; key: string }> = {
  companyName: { category: "startup_facts", key: "company_name" },
  stage: { category: "startup_facts", key: "stage" },
  market: { category: "market_knowledge", key: "industry" },
  coFounder: { category: "team_info", key: "co_founder" },
  biggestChallenge: { category: "challenges", key: "biggest_challenge" },
  ninetyDayGoal: { category: "goals", key: "90_day_goal" },
  productStatus: { category: "product_details", key: "product_status" },
  revenueStatus: { category: "metrics", key: "revenue_status" },
  teamSize: { category: "team_info", key: "team_size" },
  fundingStatus: { category: "startup_facts", key: "funding_status" },
};

export async function persistExtractedFacts(
  userId: string,
  facts: ExtractedFacts
): Promise<number> {
  const entries = Object.entries(facts).filter(([, v]) => v != null && v.length > 0);
  if (entries.length === 0) return 0;

  let stored = 0;
  for (const [key, value] of entries) {
    const mapping = FACT_TO_SEMANTIC[key as keyof ExtractedFacts];
    if (!mapping) continue;
    try {
      await storeFact(userId, mapping.category, mapping.key, { value }, {
        confidence: 1.0,
        source: "chat_extraction",
      });
      stored++;
    } catch (error) {
      console.warn(`[Active Memory] Failed to store fact ${key}:`, error);
    }
  }
  return stored;
}
```

This uses the existing `storeFact` which does UPSERT on (user_id, category, key), so new values overwrite old ones automatically.
