<task type="auto">
  <name>Task 1: Report generation orchestrator — lib/report/generate-report.ts</name>
  <files>lib/report/generate-report.ts</files>
  <action>
Create `lib/report/generate-report.ts` that exports `generateReport(userId: string)`.

This function orchestrates the full pipeline:

1. **Aggregate data:** `const aggregated = await aggregateReportData(userId)` (from `@/lib/report/aggregator`)
2. **Synthesize via AI:** `const synthesis = await synthesizeReport(aggregated)` (from `@/lib/report/synthesizer`)
   - `synthesis` returns `{ reportData, bonusSteps, usage, modelId, generationMs }` (SynthesisResult type)
3. **Build step snapshot:** Create a `Record<string, string>` from the aggregated data for diffing. Map each section's answers into a flat key-value: `{ "step-1-idea-validation": "Their answer...", ... }`. Use `aggregated.sections` to build this — iterate sections, then steps within sections, using the step ID as key and the distilled/raw answer as value.
4. **Get next version:** `const version = await getNextVersion(userId)` (from `@/lib/db/founder-reports`)
5. **Create DB record:** `const report = await createReport({ userId, version, reportData: synthesis.reportData, stepSnapshot, status: "pending" })` (from `@/lib/db/founder-reports`)
6. **Trigger PDF generation:**
   ```typescript
   import { generateFounderReportPdf } from "@/trigger/generate-founder-report-pdf";
   await generateFounderReportPdf.trigger({
     reportId: report.id,
     userId,
     version,
     reportData: synthesis.reportData,
   });
   ```
7. **Return:** `{ reportId: report.id, version, status: "pending" }`

**Return type:**
```typescript
interface GenerateReportResult {
  reportId: string;
  version: number;
  status: "pending";
}
```

**Error handling:**
- If aggregation fails (no journey data), throw a descriptive error: "No journey data found for report generation"
- If synthesis fails, let it propagate (will be caught by API route)
- If createReport fails, let it propagate
- If trigger fails after DB creation, update the report status to "failed" and re-throw

**Do NOT:**
- Import renderToBuffer or any PDF code — that's the Trigger task's job
- Pass bonusSteps to createReport — they're not in ReportData, store them separately in a future phase if needed
- Call `triggerAndWait` — use `.trigger()` for fire-and-forget

Create `app/api/reports/generate/route.ts` with a POST handler.

**Auth:** Use `requireAuth()` from `@/lib/auth` (same pattern as `app/api/dashboard/export/route.ts`).

**Handler logic:**
```typescript
import { NextResponse } from "next/server";
import { requireAuth } from "@/lib/auth";
import { generateReport } from "@/lib/report/generate-report";

export async function POST() {
  try {
    const userId = await requireAuth();
    const result = await generateReport(userId);

    return NextResponse.json(result, { status: 202 });
  } catch (error) {
    if (error instanceof Response) {
      return error; // Auth redirect
    }

    console.error("[Reports Generate API] Error:", error);

    const message = error instanceof Error ? error.message : "Failed to generate report";
    return NextResponse.json(
      { success: false, error: message },
      { status: 500 }
    );
  }
}
```

**Response shape (202):**
```json
{
  "reportId": "uuid",
  "version": 1,
  "status": "pending"
}
```

**Do NOT:**
- Add a GET handler (status polling will be a separate endpoint in Phase 94)
- Wait for PDF generation to complete — return 202 immediately
- Accept any request body — userId comes from auth, everything else is derived
