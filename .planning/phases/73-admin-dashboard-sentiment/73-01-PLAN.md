  <files>lib/feedback/sentiment.ts, lib/feedback/sentiment-aggregator.ts</files>
  <action>
Create `lib/feedback/sentiment.ts`:

1. Define `SentimentResult` type:
   ```ts
   export interface SentimentResult {
     label: 'positive' | 'neutral' | 'negative' | 'frustrated'
     confidence: number  // 0-1
     isCoachingDiscomfort: boolean  // true when negative sentiment stems from FRED challenging assumptions (working as designed)
   }
   ```

2. Export `extractSentiment(userMessage: string, fredResponse: string): Promise<SentimentResult>`:
   - Use the Vercel AI SDK `generateObject` with a small/fast model (use `getModelForTier('free', 'chat')` to get the mini model -- this is a background task, not user-facing).
   - Schema should ask the model to classify:
     a) Overall sentiment of the user message (positive/neutral/negative/frustrated)
     b) Confidence score 0-1
     c) Whether negative sentiment is "coaching discomfort" -- i.e., the user is reacting to FRED challenging their assumptions, which is FRED working as designed. Examples: "That's harsh", "I don't want to hear that", "You're being too direct" when FRED gave valid tough-love feedback.
   - Import `z` from zod for the schema.
   - Wrap in try/catch, return `{ label: 'neutral', confidence: 0, isCoachingDiscomfort: false }` on failure.
   - The function must be fast (<200ms) and non-blocking to caller.

3. Export `extractSentimentFromText(text: string): SentimentResult` (sync, heuristic-based fallback):
   - Simple keyword-based sentiment for when LLM call is not worth it (e.g., very short messages).
   - Check for frustrated patterns: "frustrated", "annoyed", "waste of time", "useless", "terrible"
   - Check for negative: "bad", "wrong", "not helpful", "disappointed", "confused"
   - Check for positive: "thanks", "great", "helpful", "amazing", "love"
   - Default to neutral with confidence 0.3.

Create `lib/feedback/sentiment-aggregator.ts`:

1. Export `aggregateSessionSentiment(sessionId: string, userId: string)`:
   - Query all `feedback_signals` for the session where `signal_type = 'sentiment'`.
   - Compute weighted average of `sentiment_score` values.
   - Determine trend: 'improving' | 'stable' | 'degrading' | 'spike_negative'.
   - Call `detectSentimentSpike` to check for sharp degradation.
   - Call `upsertFeedbackSession` from `@/lib/db/feedback` with computed values.
   - If spike detected, set `flagged: true` and `flag_reason` describing the spike.

2. Export `detectSentimentSpike(scores: number[]): { spiked: boolean; reason: string }`:
   - A spike is when the last 3 messages average sentiment is < -0.5 AND the previous 3 messages were > 0 (sharp drop).
   - Also flag if any single message has sentiment < -0.8 with confidence > 0.7.
   - Return descriptive reason: "Sharp sentiment degradation: avg dropped from X to Y over last N messages"

3. Map sentiment labels to numeric scores for aggregation: positive=1, neutral=0, negative=-0.5, frustrated=-1.
