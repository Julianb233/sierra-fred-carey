<task type="auto">
  <name>Task 1: Create admin feedback DB queries and API routes</name>
  <files>lib/db/feedback-admin.ts, app/api/admin/feedback/route.ts, app/api/admin/feedback/stats/route.ts</files>
  <action>
Create `lib/db/feedback-admin.ts`:

1. Export `queryFeedbackSignalsAdmin(filters)`:
   - Accept a filters object: `{ dateFrom?: string, dateTo?: string, channel?: string, rating?: number, category?: string, tier?: string, userId?: string, page?: number, limit?: number }`
   - Build a Supabase query on `feedback_signals` table with `.select('*')`.
   - Apply each filter conditionally: `.gte('created_at', dateFrom)`, `.lte('created_at', dateTo)`, `.eq('channel', channel)`, etc.
   - Apply `.order('created_at', { ascending: false })`.
   - Apply pagination: `.range(offset, offset + limit - 1)` where offset = (page - 1) * limit. Default limit=50, page=1.
   - Use `{ count: 'exact' }` in the select to get total count for pagination.
   - Use `createServiceClient()` (admin queries bypass RLS).
   - Return `{ data: FeedbackSignal[], total: number }`.

2. Export `getFeedbackStats(dateFrom?: string, dateTo?: string)`:
   - Query `feedback_signals` with date range filter.
   - Compute: total count, positive count (rating=1), negative count (rating=-1), sentiment count (signal_type='sentiment').
   - Group by category using a separate query: `.select('category')` then count in JS.
   - Group by day for daily volume: query `created_at` and bucket by date in JS.
   - Return `{ totalSignals, positiveCount, negativeCount, sentimentCount, categoryDistribution: Record<string, number>, dailyVolume: Array<{ date: string, count: number }> }`.

Create `app/api/admin/feedback/route.ts`:

1. GET handler that:
   - Calls `requireAdminRequest(request)` from `@/lib/auth/admin` -- return 401 if denied.
   - Parses query params from `request.nextUrl.searchParams`: dateFrom, dateTo, channel, rating (parse to number), category, tier, userId, page, limit.
   - Calls `queryFeedbackSignalsAdmin(filters)`.
   - Returns JSON `{ signals, total, page, limit }`.

Create `app/api/admin/feedback/stats/route.ts`:

1. GET handler that:
   - Calls `requireAdminRequest(request)`.
   - Parses dateFrom, dateTo from query params.
   - Calls `getFeedbackStats(dateFrom, dateTo)`.
   - Returns the stats object as JSON.

Both API routes: import `{ requireAdminRequest }` from `@/lib/auth/admin`. Follow the existing admin API pattern (see how other admin routes like `/api/admin/dashboard` work).
