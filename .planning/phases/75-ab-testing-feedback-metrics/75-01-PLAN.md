   JOIN ab_experiments e ON v.experiment_id = e.id

**lib/statistics/significance.ts** — Pure TypeScript implementations of chi-squared and Welch's t-test.

No external dependencies. All CDF approximations implemented inline.

```typescript
// Types
export interface ChiSquaredResult {
  chiSquared: number
  degreesOfFreedom: number
  pValue: number
  significant: boolean // true if p < alpha
  alpha: number
}

export interface TTestResult {
  tStatistic: number
  degreesOfFreedom: number
  pValue: number
  significant: boolean
  alpha: number
  meanDifference: number
  confidenceInterval: [number, number] // 95% CI for mean difference
}

// Constants
const MIN_SAMPLE_SIZE = 500 // REQ-A2: minimum sessions per variant
const DEFAULT_ALPHA = 0.05 // 95% confidence
```

Implement these functions:

1. **`chiSquaredTest(observed1: [number, number], observed2: [number, number], alpha?: number): ChiSquaredResult`**
   - Input: Two pairs of [success, failure] counts (e.g., [thumbsUp, thumbsDown] for each variant)
   - Uses 2x2 contingency table with Yates' continuity correction
   - Formula: `chi2 = sum((|O-E| - 0.5)^2 / E)` for each cell
   - 1 degree of freedom for 2x2 table
   - p-value from chi-squared CDF approximation (use the incomplete gamma function approach)
   - Returns ChiSquaredResult

2. **`welchTTest(sample1: { mean: number, variance: number, n: number }, sample2: { mean: number, variance: number, n: number }, alpha?: number): TTestResult`**
   - Welch's t-test (does NOT assume equal variance)
   - t = (mean1 - mean2) / sqrt(var1/n1 + var2/n2)
   - Degrees of freedom via Welch-Satterthwaite: `df = (var1/n1 + var2/n2)^2 / ((var1/n1)^2/(n1-1) + (var2/n2)^2/(n2-1))`
   - p-value from Student's t CDF (two-tailed)
   - 95% confidence interval for mean difference
   - Returns TTestResult

3. **`isSignificant(pValue: number, alpha?: number): boolean`** — Simple helper

4. **`meetsMinimumSampleSize(n1: number, n2: number, minimum?: number): boolean`** — Validates both groups meet the minimum (default 500)

5. **Internal CDF helpers** (not exported):
   - `chiSquaredCDF(x: number, k: number): number` — Using regularized incomplete gamma function
   - `tDistributionCDF(t: number, df: number): number` — Using regularized incomplete beta function
   - `gammaLn(x: number): number` — Log-gamma via Lanczos approximation
   - `regularizedIncompleteGamma(a: number, x: number): number`
   - `regularizedIncompleteBeta(a: number, b: number, x: number): number`

Use Lanczos approximation for gamma function (accurate to ~15 digits). For the incomplete beta function, use the continued fraction representation (Lentz's method).

**lib/statistics/significance.test.ts** — Unit tests using Vitest.

Test cases:
1. Chi-squared with known values: `chiSquaredTest([100, 50], [80, 70])` — verify against known chi-squared table
2. Chi-squared with identical distributions returns non-significant

4. Welch's t-test with known means/variances — verify against known t-table

6. `meetsMinimumSampleSize(500, 500)` returns true
7. `meetsMinimumSampleSize(499, 500)` returns false
8. Edge cases: zero counts, single observation, very large samples

Types:
```typescript
export interface VariantFeedbackMetrics {
  variantId: string
  variantName: string
  experimentName: string
  // Thumbs metrics
  thumbsUpCount: number
  thumbsDownCount: number
  thumbsRatio: number // thumbsUp / (thumbsUp + thumbsDown), 0-1
  // Sentiment metrics
  avgSentimentScore: number // -1 to 1
  sentimentStdDev: number
  sentimentSampleSize: number
  // Session metrics
  totalSessions: number
  completedSessions: number
  sessionCompletionRate: number // completed / total, 0-1
  // Feedback volume
  totalFeedbackSignals: number
}

export interface ExperimentFeedbackComparison {
  experimentName: string
  variants: VariantFeedbackMetrics[]
  // Significance results
  thumbsSignificance: {
    chiSquared: number
    pValue: number
    significant: boolean
    meetsMinimumSample: boolean
  } | null
  sentimentSignificance: {
    tStatistic: number
    pValue: number
    significant: boolean
    meanDifference: number
    confidenceInterval: [number, number]
    meetsMinimumSample: boolean
  } | null
  sessionCompletionSignificance: {
    chiSquared: number
    pValue: number
    significant: boolean
    meetsMinimumSample: boolean
  } | null
  // Winner detection
  feedbackWinner: string | null // variant name with best combined feedback metrics
  winnerConfidence: number | null
}
```

Functions:

1. **`getVariantFeedbackMetrics(variantId: string): Promise<VariantFeedbackMetrics | null>`**
   - Query feedback_signals WHERE variant_id = variantId
   - Compute thumbs counts, sentiment averages, session completion
   - Use the variant_feedback_summary view if available, fallback to direct query
   - Return null if no feedback data exists for this variant

2. **`getExperimentFeedbackComparison(experimentName: string): Promise<ExperimentFeedbackComparison>`**
   - Get all variants for the experiment
   - Call getVariantFeedbackMetrics for each
   - Run chiSquaredTest on thumbs_up vs thumbs_down between control and treatment
   - Run welchTTest on sentiment scores between control and treatment
   - Run chiSquaredTest on session completion between control and treatment
   - Check meetsMinimumSampleSize for each test
   - Determine feedbackWinner: variant with highest combined score (0.4 * thumbsRatio + 0.4 * normalizedSentiment + 0.2 * sessionCompletionRate), only if at least one significance test passes

3. **`getVariantFeedbackMetricsBatch(variantIds: string[]): Promise<Map<string, VariantFeedbackMetrics>>`**
   - Batch query for efficiency — single SQL query for all variants
   - Used by the enhanced getVariantStats

Key design decisions:
- Control variant identified by name "control" (existing convention from ab-test-metrics.ts)
- For experiments with >2 variants, compare each treatment to control
- If no control variant, compare top 2 by thumbs ratio
- All queries use service role (admin context) — these are admin-only metrics
