**lib/feedback/pre-registration.ts** — Pre-registration template for experiments.

Types:
```typescript
export interface PreRegistration {
  // Required fields
  hypothesis: string // "Changing FRED's coaching tone to be more empathetic will increase thumbs-up ratio by 10%"
  primaryMetric: PreRegistrationMetric
  // Optional but recommended
  secondaryMetrics: PreRegistrationMetric[]
  minimumSampleSize: number // per variant, default 500 (REQ-A2)
  expectedDuration: string // human-readable, e.g., "2 weeks"
  expectedDurationDays: number // numeric for automatic end date
  // Context
  rationale: string // Why this experiment matters
  segmentFilter?: string // Optional: specific user segment (tier, stage, etc.)
  // Status tracking
  registeredAt: string // ISO timestamp
  registeredBy: string // user ID
}

export interface PreRegistrationMetric {
  name: string // e.g., "thumbsUpRatio", "avgSentimentScore", "sessionCompletionRate"
  type: 'binary' | 'continuous' // determines which stat test to use
  direction: 'higher_is_better' | 'lower_is_better'
  minimumDetectableEffect: number // e.g., 0.1 for 10% improvement
}

// Preset metrics for common experiments
export const PRESET_METRICS: Record<string, PreRegistrationMetric> = {
  thumbsUpRatio: {
    name: 'Thumbs-Up Ratio',
    type: 'binary',
    direction: 'higher_is_better',
    minimumDetectableEffect: 0.05,
  },
  avgSentimentScore: {
    name: 'Average Sentiment Score',
    type: 'continuous',
    direction: 'higher_is_better',
    minimumDetectableEffect: 0.1,
  },
  sessionCompletionRate: {
    name: 'Session Completion Rate',
    type: 'binary',
    direction: 'higher_is_better',
    minimumDetectableEffect: 0.05,
  },
  errorRate: {
    name: 'Error Rate',
    type: 'binary',
    direction: 'lower_is_better',
    minimumDetectableEffect: 0.02,
  },
  avgLatency: {
    name: 'Average Latency',
    type: 'continuous',
    direction: 'lower_is_better',
    minimumDetectableEffect: 100, // ms
  },
}
```

Functions:

1. **`validatePreRegistration(reg: Partial<PreRegistration>): { valid: boolean, errors: string[] }`**
   - hypothesis must be non-empty, at least 20 characters
   - primaryMetric must have name, type, and direction
   - minimumSampleSize must be >= 100 (warn if < 500)
   - expectedDurationDays must be > 0
   - Return validation errors array

2. **`createPreRegisteredExperiment(params: { name: string, description: string, variants: Array<{ variantName: string, promptId?: string, configOverrides?: Record<string, unknown>, trafficPercentage: number }>, preRegistration: PreRegistration, userId: string }): Promise<string>`**
   - Validate pre-registration first
   - Call existing `createExperiment()` from `@/lib/ai/ab-testing`
   - Then update the experiment's metadata JSONB to include the pre-registration:
     ```sql
     UPDATE ab_experiments SET metadata = jsonb_set(COALESCE(metadata, '{}'), '{preRegistration}', $1::jsonb) WHERE id = $2
     ```
   - Note: ab_experiments may not have a metadata column yet. If not, the migration in Task... we need to add it. Check if it exists first.
   - ALTERNATIVE: Store pre-registration in a separate `metadata` column. Since ab_experiments doesn't have a metadata JSONB column, add one via the same migration from 75-01 or create a small follow-up migration here.
   - Return experiment ID

3. **`getPreRegistration(experimentId: string): Promise<PreRegistration | null>`**
   - Query ab_experiments metadata->preRegistration
   - Return null if no pre-registration exists

**app/api/admin/ab-tests/route.ts** — API for experiment listing and creation.

```typescript
import { NextRequest, NextResponse } from "next/server"
import { requireAdmin } from "@/lib/auth/admin"
import { getActiveExperiments } from "@/lib/ai/ab-testing"
import { getVariantStatsWithFeedback } from "@/lib/ai/ab-testing"
import { getExperimentFeedbackComparison } from "@/lib/feedback/experiment-metrics"
import { createPreRegisteredExperiment, validatePreRegistration } from "@/lib/feedback/pre-registration"
import { sql } from "@/lib/db/supabase-sql"
```

GET handler:
1. Call `requireAdmin()` to verify admin access
2. Query all experiments (active and completed, ordered by start_date DESC):
   ```sql
   SELECT id, name, description, is_active, start_date, end_date, metadata
   FROM ab_experiments
   ORDER BY start_date DESC
   LIMIT 50
   ```
3. For each experiment, call `getVariantStatsWithFeedback(name)` to get variant metrics with feedback
4. For active experiments, also call `getExperimentFeedbackComparison(name)` for significance data
5. Map to response shape matching what the admin page expects (ABTest interface)
6. Include pre-registration data from metadata column
7. Return JSON array

POST handler:
1. Call `requireAdmin()` to verify admin access
2. Parse request body: `{ name, description, variants, preRegistration }`
3. Validate pre-registration via `validatePreRegistration()`
4. If validation fails, return 400 with errors
5. Call `createPreRegisteredExperiment()` with user ID from auth
6. Return 201 with experiment ID
