Create a Supabase migration with two tables:

1. **`fewshot_examples`** — stores positive and negative few-shot examples from feedback:
   ```sql
   CREATE TABLE IF NOT EXISTS fewshot_examples (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     signal_id UUID NOT NULL REFERENCES feedback_signals(id) ON DELETE CASCADE,
     user_id UUID NOT NULL,
     topic TEXT NOT NULL,                    -- coaching topic (fundraising, strategy, etc.)
     example_type TEXT NOT NULL CHECK (example_type IN ('positive', 'negative')),
     user_message TEXT NOT NULL,             -- the founder's message
     assistant_response TEXT NOT NULL,       -- FRED's response
     category TEXT,                          -- feedback category (from thumbs-down)
     comment TEXT,                           -- user's comment explaining rating
     user_tier TEXT NOT NULL DEFAULT 'free', -- for tier weighting
     weight NUMERIC(3,1) NOT NULL DEFAULT 1.0,
     metadata JSONB NOT NULL DEFAULT '{}',
     created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
     expires_at TIMESTAMPTZ                 -- GDPR: auto-expire with signal
   );

   CREATE INDEX idx_fewshot_topic_type ON fewshot_examples(topic, example_type);
   CREATE INDEX idx_fewshot_signal ON fewshot_examples(signal_id);
   CREATE INDEX idx_fewshot_expires ON fewshot_examples(expires_at) WHERE expires_at IS NOT NULL;
   ```

2. **`prompt_patches`** — stores generated prompt patches with approval workflow:
   ```sql
   CREATE TABLE IF NOT EXISTS prompt_patches (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     title TEXT NOT NULL,
     content TEXT NOT NULL,                  -- the supplemental prompt text
     topic TEXT,                             -- coaching topic this targets
     source TEXT NOT NULL CHECK (source IN ('feedback', 'ab_test', 'manual')),
     source_id TEXT,                         -- feedback_insight.id or cluster hash
     source_signal_ids UUID[] DEFAULT '{}',  -- feedback signal IDs that triggered this
     status TEXT NOT NULL DEFAULT 'draft' CHECK (status IN ('draft', 'approved', 'active', 'testing', 'rejected', 'archived')),
     version INTEGER NOT NULL DEFAULT 1,
     parent_patch_id UUID REFERENCES prompt_patches(id),
     experiment_id UUID REFERENCES ab_experiments(id),  -- if launched as A/B test
     approved_by UUID,
     approved_at TIMESTAMPTZ,
     activated_at TIMESTAMPTZ,
     deactivated_at TIMESTAMPTZ,
     performance_metrics JSONB DEFAULT '{}', -- thumbs improvement, etc.
     metadata JSONB NOT NULL DEFAULT '{}',
     created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
     updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
   );

   CREATE INDEX idx_prompt_patches_status ON prompt_patches(status);
   CREATE INDEX idx_prompt_patches_topic ON prompt_patches(topic) WHERE topic IS NOT NULL;
   CREATE INDEX idx_prompt_patches_source ON prompt_patches(source, source_id);
   ```

**lib/rlhf/few-shot-store.ts** — CRUD operations for few-shot examples with tier-weighted retrieval.

Imports:
- `sql` from `@/lib/db/supabase-sql`
- `TIER_WEIGHTS` from `@/lib/feedback/constants`
- Types from `@/lib/rlhf/types`
- `logger` from `@/lib/logger`

Functions:

1. **`storeFewShotExample(insert: FewShotExampleInsert): Promise<FewShotExample>`**
   - INSERT into fewshot_examples
   - Auto-set weight from TIER_WEIGHTS based on userTier
   - Return the created row

2. **`createFewShotFromSignal(params: { signalId: string, userId: string, topic: string, exampleType: ExampleType, userMessage: string, assistantResponse: string, category?: string, comment?: string, userTier: 'free'|'pro'|'studio', expiresAt?: string }): Promise<FewShotExample>`**
   - Convenience wrapper that builds FewShotExampleInsert and calls storeFewShotExample
   - Validates topic is non-empty
   - Deduplicates: checks if a fewshot_example already exists for this signal_id, returns existing if found

3. **`getFewShotExamplesForTopic(topic: string, exampleType: ExampleType, limit?: number): Promise<FewShotExample[]>`**
   - Query fewshot_examples WHERE topic = topic AND example_type = exampleType
   - Filter out expired examples (expires_at IS NULL OR expires_at > NOW())
   - ORDER BY weight DESC, created_at DESC
   - Default limit 10
   - Return mapped rows

4. **`getTopFewShotExamples(topic: string, options?: { positiveLimit?: number, negativeLimit?: number }): Promise<{ positive: FewShotExample[], negative: FewShotExample[] }>`**
   - Calls getFewShotExamplesForTopic twice (once for positive, once for negative)
   - Default 5 positive, 3 negative
   - These are used as context for the LLM prompt patch generator

5. **`pruneStaleFewShots(): Promise<number>`**
   - DELETE FROM fewshot_examples WHERE expires_at < NOW()
   - Returns count of deleted rows
   - Called by daily cleanup job

Schema for structured output:
```typescript
const patchSchema = z.object({
  title: z.string().describe("Short descriptive title for the patch, e.g. 'Reduce generic responses for fundraising questions'"),
  content: z.string().describe("The supplemental prompt instruction text. 2-5 sentences. Specific and actionable."),
  topic: z.string().nullable().describe("Coaching topic this targets, or null if general"),
  rationale: z.string().describe("Why this patch addresses the feedback pattern"),
  expectedImprovement: z.string().describe("What metric should improve and estimated magnitude"),
  confidence: z.enum(["low", "medium", "high"]).describe("Confidence that this patch will improve the metric"),
})
```

Functions:

1. **`generatePromptPatch(request: PatchGenerationRequest): Promise<PatchGenerationResult>`**
   - Construct a prompt with:
     - The cluster theme and description
     - Signal comments (max 20, truncated to 200 chars each)
     - Positive few-shot examples (show what FRED did well)
     - Negative few-shot examples (show what FRED should avoid)
     - Any existing active patches for this topic (to avoid duplication)
   - Use `generateObject` with the patchSchema
   - Use `getModelForTier('free', 'structured')` (cost-efficient for internal generation)
   - Return the PatchGenerationResult

2. **`generatePatchFromCluster(params: { clusterTheme: string, clusterDescription: string, category: string | null, severity: string, signalIds: string[], signalComments: string[] }): Promise<PromptPatch>`**
   - Orchestrator function that:
     a. Gets few-shot examples for the cluster's topic/category using getTopFewShotExamples
     b. Gets existing active patches for the same topic
     c. Calls generatePromptPatch with full context
     d. Inserts the result into prompt_patches table with status='draft'
     e. Returns the created PromptPatch row
   - Never auto-activates — all patches start as draft

3. **`savePatchToDB(patch: Omit<PromptPatch, 'id' | 'createdAt' | 'updatedAt'>): Promise<PromptPatch>`**
   - INSERT into prompt_patches
   - Return created row with DB-generated fields

4. **`getActivePatches(topic?: string): Promise<PromptPatch[]>`**
   - Query prompt_patches WHERE status = 'active'
   - Optionally filter by topic
   - Used to check for existing patches before generating new ones
