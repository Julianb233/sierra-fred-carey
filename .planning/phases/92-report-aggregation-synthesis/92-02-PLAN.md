Create `lib/report/report-schema.ts` with Zod schemas that match the `ReportData` and `ReportSection` interfaces from `types/report.ts`.

```typescript
import { z } from "zod"

export const reportSectionSchema = z.object({
  id: z.string().describe("Section identifier matching the section ID from step mapping"),
  title: z.string().describe("Display title for this section"),
  synthesized: z.string().min(100).describe("FRED's narrative synthesis of this section - 2-4 paragraphs grounded in the founder's actual answers"),
  highlights: z.array(z.string()).min(2).max(5).describe("Key strength highlights from this section - specific to this founder, never generic"),
  stepIds: z.array(z.string()).describe("Report step IDs that contributed to this section"),
})

export const reportDataSchema = z.object({
  executiveSummary: z.string().min(150).max(800).describe("FRED's 3-5 sentence synthesis of the entire business model - specific, grounded, celebratory but honest"),
  founderName: z.string(),
  companyName: z.string(),
  generatedAt: z.string(),
  sections: z.array(reportSectionSchema).length(5),
  fredSignoff: z.string().min(50).describe("FRED's personal closing message to this founder - warm, specific, forward-looking"),
  bonusSteps: z.array(z.object({
    title: z.string().describe("Short action-oriented title"),
    description: z.string().min(50).describe("Why this step matters for THIS specific business"),
    rationale: z.string().describe("Which answers led FRED to suggest this"),
  })).min(1).max(2).describe("Personalized next steps FRED recommends based on the specific gaps or opportunities in this founder's answers"),
})
```

The `bonusSteps` field is added beyond ReportData — the synthesizer will extract it and pass it alongside ReportData. Add a `SynthesisOutput` type that extends ReportData with bonusSteps.

NOTE: The `bonusSteps` is NOT part of ReportData stored in the DB. The synthesizer returns both `reportData: ReportData` and `bonusSteps` separately. The bonusSteps can be stored in a separate field or appended to fredSignoff later. For now, the schema captures both so a single AI call produces everything.

  <name>Task 2: Create synthesizer.ts — FRED AI synthesis with anti-sycophancy guardrails</name>
  <files>lib/report/synthesizer.ts</files>
  <action>
Create `lib/report/synthesizer.ts` that transforms `AggregatedReportInput` into `ReportData` via a single batched AI call.

Function signature:
```typescript
export async function synthesizeReport(
  input: AggregatedReportInput
): Promise<{ reportData: ReportData; bonusSteps: BonusStep[]; usage: { promptTokens: number; completionTokens: number }; modelId: string; generationMs: number }>
```

Implementation:

1. **Build the synthesis prompt** — a single system prompt containing:
   - FRED's identity (from FRED_IDENTITY): name, tagline, years of experience
   - FRED's voice rules (from FRED_COMMUNICATION_STYLE): direct, no-BS, warm but honest, storytelling from experience
   - Anti-sycophancy rules (critical — this is RGEN-05):
     ```
     ANTI-SYCOPHANCY RULES (MANDATORY):
     - Every claim must reference the founder's actual answer. If they said "I plan to acquire customers through Instagram ads", write "Your plan to acquire customers through Instagram ads..." — not "Your strong marketing strategy..."
     - NEVER use these phrases unless directly supported by evidence in the answers:
       "passionate team", "strong market opportunity", "innovative solution", "disruptive technology",
       "game-changing", "revolutionary", "best-in-class", "world-class", "cutting-edge",
       "synergy", "leverage", "robust growth", "impressive traction"
     - If a section has weak or missing answers, say so honestly: "This area needs more development" — do not invent strengths
     - Positive means SPECIFIC: "Your pricing at $29/month positions you below competitor X's $49 while covering your $12 CAC" — not "Your pricing strategy is well-thought-out"
     - Temperature is set to 0.3 for grounded, deterministic output
     ```
   - The 5 section descriptions and what each section should cover
   - Instructions for executive summary (3-5 sentences, captures the full business model)
   - Instructions for bonus steps (analyze gaps/opportunities, suggest 1-2 personalized next actions)
   - Instructions for fredSignoff (warm closing, reference specific things from the journey)

2. **Build the user prompt** — the actual founder data:
   - Founder name, company name, industry
   - For each section, list each step with its answer (or "[No answer provided]" for null)
   - Format clearly so the AI can reference specific answers

3. **Call generateStructured()** from `lib/ai/fred-client.ts`:
   - Schema: `reportDataSchema` from report-schema.ts
   - Temperature: 0.3 (anti-sycophancy, grounded output)
   - Model: "primary" (best available model for quality)
   - maxOutputTokens: 8192 (report is substantial — 5 sections + exec summary + signoff + bonus steps)
   - System prompt: the synthesis prompt built in step 1
   - User prompt: the founder data built in step 2

4. **Post-process the result:**
   - Extract `bonusSteps` from the AI output
   - Build `ReportData` (without bonusSteps) matching the `types/report.ts` interface
   - Set `generatedAt` to current ISO timestamp
   - Return `{ reportData, bonusSteps, usage, modelId, generationMs }`

5. **Measure generation time** — wrap the AI call in `performance.now()` bookends

IMPORTANT anti-sycophancy enforcement in the prompt:
- The prompt must include BAD vs GOOD examples (pattern from FRED_CORE_PROMPT v1.1.0):
  ```
  BAD: "Your impressive go-to-market strategy shows strong potential for market domination."
  GOOD: "You plan to reach customers through Instagram ads targeting women 25-35 in Austin. Your $15 CAC assumption needs validation — have you run a test campaign?"

  BAD: "Your passionate founding team brings a wealth of experience."
  GOOD: "You're a solo founder with 8 years in SaaS sales. Your weakness: no technical co-founder. Your plan to outsource development is viable but adds risk to your timeline."
  ```

Do NOT import the full FRED_CORE_PROMPT for the report synthesis — it's designed for chat, not report generation. Build a focused synthesis prompt that captures FRED's voice without the chat-specific instructions (step guidance, drift detection, reality lens gates, etc.).

Import FRED_IDENTITY, FRED_BIO, FRED_COMMUNICATION_STYLE directly from `lib/fred-brain.ts` to build the voice section.
