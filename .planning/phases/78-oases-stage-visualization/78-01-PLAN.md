**Migration SQL** (`supabase/migrations/20260308000001_oases_stage.sql`):

```sql
-- Add oases_stage column to profiles (default: clarity for new users)
ALTER TABLE profiles
  ADD COLUMN IF NOT EXISTS oases_stage TEXT NOT NULL DEFAULT 'clarity'
  CHECK (oases_stage IN ('clarity', 'validation', 'build', 'launch', 'grow'));

-- Create oases_progress table to track step completion within each stage
CREATE TABLE IF NOT EXISTS oases_progress (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  stage TEXT NOT NULL CHECK (stage IN ('clarity', 'validation', 'build', 'launch', 'grow')),
  step_id TEXT NOT NULL,
  completed_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  score NUMERIC(5,2),
  metadata JSONB DEFAULT '{}',
  UNIQUE(user_id, stage, step_id)
);

-- RLS: users can only see/modify their own progress
ALTER TABLE oases_progress ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users read own oases_progress"
  ON oases_progress FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Users insert own oases_progress"
  ON oases_progress FOR INSERT
  WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users update own oases_progress"
  ON oases_progress FOR UPDATE
  USING (auth.uid() = user_id);

-- Index for fast lookups
CREATE INDEX IF NOT EXISTS idx_oases_progress_user_stage
  ON oases_progress(user_id, stage);
```

**Progress Logic** (`lib/oases/progress.ts`):

Server-side module (uses Supabase server client). Import `createClient` from `@/lib/supabase/server`.

Functions:

```typescript
export async function getUserOasesProgress(userId: string): Promise<OasesProgress>
```
- Fetches user's `oases_stage` from profiles table
- Fetches all rows from `oases_progress` for that user
- For each stage in STAGE_ORDER:
  - Count completed steps vs total steps from STAGE_CONFIG
  - Set status: "completed" if all steps done and stage is before current, "current" if matches oases_stage, "locked" otherwise
- Compute `journeyPercentage`: (total completed steps across ALL stages / total steps across ALL stages) * 100, rounded to nearest integer
- Compute `canAdvance`: all steps in current stage are complete

```typescript
export async function advanceStage(userId: string): Promise<{ success: boolean; newStage?: OasesStage; error?: string }>
```
- Gets current progress
- If not canAdvance, return error
- If current stage is "grow", return error (already at final stage)
- Update profiles.oases_stage to next stage in STAGE_ORDER
- Return success with new stage

```typescript
export function computeJourneyPercentage(completedSteps: number, totalSteps: number): number
```
- Simple utility: Math.round((completedSteps / totalSteps) * 100)
