```typescript
export interface MemoryField<T = string> {
  value: T | null
  confidence: number  // 0-1, how certain we are this is correct
  source: 'profile' | 'onboarding' | 'conversation' | 'enrichment'
  lastUpdated: Date | null
}

export const CORE_MEMORY_FIELDS = [
  'founder_name',
  'company_name',
  'stage',
  'market',
  'co_founder',
  'biggest_challenge',
  'oases_stage',
] as const

export type CoreMemoryFieldKey = typeof CORE_MEMORY_FIELDS[number]

export interface FounderMemory {
  founder_name: MemoryField
  company_name: MemoryField
  stage: MemoryField           // e.g. "pre-seed", "seed", "series-a"
  market: MemoryField          // industry/sector
  co_founder: MemoryField      // co-founder name(s) or "solo founder"
  biggest_challenge: MemoryField
  oases_stage: MemoryField     // Oases journey stage (clarity, validation, build, launch, grow)
  // Extended fields from semantic memory
  additional: Record<string, MemoryField>
}

export interface MemoryUpdate {
  field: CoreMemoryFieldKey | string
  value: string
  confidence: number
  source: 'conversation'
}

/** Default staleness threshold in days */
export const STALENESS_THRESHOLD_DAYS = 7

/** Check if a memory field is stale (older than threshold) */
export function isStale(field: MemoryField, thresholdDays = STALENESS_THRESHOLD_DAYS): boolean {
  if (!field.lastUpdated || !field.value) return true
  const ageMs = Date.now() - field.lastUpdated.getTime()
  return ageMs > thresholdDays * 24 * 60 * 60 * 1000
}

/** Get all stale core fields from a FounderMemory */
export function getStaleFields(memory: FounderMemory): CoreMemoryFieldKey[] {
  return CORE_MEMORY_FIELDS.filter(key => {
    const field = memory[key]
    return field.value && isStale(field)
  })
}

/** Get all missing core fields (no value at all) */
export function getMissingFields(memory: FounderMemory): CoreMemoryFieldKey[] {
  return CORE_MEMORY_FIELDS.filter(key => !memory[key].value)
}
```

Create `lib/fred/active-memory.ts` -- the core engine that assembles and updates founder memory.

**buildActiveFounderMemory(userId: string, hasPersistentMemory: boolean): Promise&lt;FounderMemory&gt;**

This function merges multiple data sources into a single FounderMemory:

1. Load profile from `profiles` table (via Supabase service client):
   - `name` -> `founder_name` (confidence 0.9, source 'profile')
   - `company_name` -> `company_name` (confidence 0.9, source 'profile')
   - `stage` -> `stage` (confidence 0.8, source 'profile')
   - `industry` -> `market` (confidence 0.8, source 'profile')
   - `co_founder` -> `co_founder` (confidence 0.9, source 'profile')
   - `challenges[0]` -> `biggest_challenge` (confidence 0.7, source 'profile')
   - `oases_stage` -> `oases_stage` (confidence 1.0, source 'profile')

2. Load semantic memory facts (via `getAllUserFacts` from `@/lib/db/fred-memory`) if hasPersistentMemory is true. Override profile fields with higher-confidence facts where available. Map known semantic categories:
   - `startup_facts/company_name` -> company_name
   - `startup_facts/market` or `startup_facts/industry` -> market
   - `challenges/primary_constraint` or `challenges/biggest_challenge` -> biggest_challenge
   - `team_info/co_founder` -> co_founder
   - Any other facts -> `additional` record

3. Load enrichment_data from profile's JSONB column as fallback for missing fields.

4. For each field: if value exists, set confidence based on source recency. If `updated_at` column is available, use it for `lastUpdated`.

**formatMemoryBlock(memory: FounderMemory): string**

Formats the FounderMemory into a prompt block for injection. Structure:

```
## ACTIVE FOUNDER CONTEXT

**Founder:** {name}
**Company:** {company_name}
**Stage:** {stage}
**Market:** {market}
**Co-Founder:** {co_founder}
**Biggest Challenge:** {biggest_challenge}
**Journey Stage:** {oases_stage}

CRITICAL INSTRUCTION: You MUST reference at least one of the above details in EVERY response. Never give generic advice. Address this founder by context: "Since {company_name} is targeting {market}..." or "Given your challenge with {biggest_challenge}...". If context above is missing for a field, ask the founder about it rather than guessing.

{If stale fields exist:}
**Stale Context (confirm these):** The following were last updated over 7 days ago. Naturally confirm them: {stale_field_list}. Example: "Last time we talked, your biggest challenge was {value}. Is that still the case?"

{If missing core fields exist:}
**Missing Context:** {missing_field_list}. Weave these into the next 1-2 exchanges naturally. Do NOT ask all at once.
```

**extractMemoryUpdates(userMessage: string, assistantResponse: string): Promise&lt;MemoryUpdate[]&gt;**

Uses the `MEMORY_EXTRACTION_PROMPT` from `@/lib/ai/memory-extraction-prompt.ts` + the `generate` function from `@/lib/ai/fred-client.ts` to extract facts from the user message. Parse result with `parseExtractionResult`. Use temperature 0.2 and maxOutputTokens 256 for fast, deterministic extraction.

**persistMemoryUpdates(userId: string, updates: MemoryUpdate[]): Promise&lt;void&gt;**

For each update:
- If field is a core profile column (founder_name -> name, company_name, co_founder, stage mapped to profiles.stage, market mapped to profiles.industry): UPDATE the profiles table directly.
- If field is a semantic memory field: use `storeFact` from `@/lib/db/fred-memory` with category mapping:
  - biggest_challenge -> category 'challenges', key 'biggest_challenge'
  - company_name -> category 'startup_facts', key 'company_name'
  - market -> category 'startup_facts', key 'market'
  - co_founder -> category 'team_info', key 'co_founder'
- Store both (profile column AND semantic fact) for redundancy -- profile is source of truth, semantic fact enables search.

Important: This function is designed to be called fire-and-forget (non-blocking after chat response). Wrap in try/catch, log warnings on failure, never throw.

Use `createServiceClient` from `@/lib/supabase/server` for all DB access. Follow existing patterns in `context-builder.ts`.
