<objective>
Verify and test the Stripe webhook BUILDER tier flow end-to-end, focusing on the critical C3 pitfall (webhook event ordering race), and unconditionally harden the DB fallback in resolveUserIdFromSubscription.

Purpose: When a founder pays $39 for the BUILDER tier, Stripe fires multiple webhook events (`checkout.session.completed`, `customer.subscription.created`, `customer.subscription.updated`). If `subscription.updated` arrives before `session.completed`, the handler must still resolve the userId and update the tier. This plan tests that the existing webhook code handles BUILDER correctly and hardens the DB fallback path regardless of test outcomes.

Output: Webhook integration test file proving BUILDER checkout works, including the race condition scenario. Hardened resolveUserIdFromSubscription with guaranteed getSubscriptionByCustomerId fallback.
</objective>
