Step 1: Check if NEXT_PUBLIC_STRIPE_BUILDER_PRICE_ID already exists in `.env.local`:
```bash
grep 'NEXT_PUBLIC_STRIPE_BUILDER_PRICE_ID' .env.local
```

Step 2: Check how `lib/stripe/config.ts` references the BUILDER price ID. Look at the PLANS object to see if it reads from `process.env.NEXT_PUBLIC_STRIPE_BUILDER_PRICE_ID` or has a hardcoded value.

Step 3: If the env var is NOT set, use the Stripe CLI to check existing products/prices:
```bash
npx stripe products list --limit 20 2>/dev/null || echo "Stripe CLI not available"
npx stripe prices list --limit 20 --lookup-keys builder 2>/dev/null || echo "No prices found"
```

If Stripe CLI is not available, check if there is a STRIPE_SECRET_KEY in .env.local and use curl:
```bash
source .env.local
curl -s https://api.stripe.com/v1/prices?limit=20 -u "$STRIPE_SECRET_KEY:" | grep -A5 '"nickname".*[Bb]uilder'
```

Step 4: If a BUILDER price already exists in Stripe, grab its price_id (starts with `price_`). If no BUILDER price exists, create one:
```bash
# Create product
curl -s https://api.stripe.com/v1/products -u "$STRIPE_SECRET_KEY:" \
  -d name="Builder" \
  -d description="Sahara Builder tier - $39/month" \
  -d metadata[tier]=builder

# Create price on that product
curl -s https://api.stripe.com/v1/prices -u "$STRIPE_SECRET_KEY:" \
  -d product=prod_XXXXX \
  -d unit_amount=3900 \
  -d currency=usd \
  -d "recurring[interval]"=month \
  -d nickname="Builder Monthly" \
  -d lookup_key=builder
```

Step 5: Add the price ID to `.env.local`:
```
NEXT_PUBLIC_STRIPE_BUILDER_PRICE_ID=price_XXXXX
```

Step 6: Verify `lib/stripe/config.ts` reads from this env var. If PLANS.BUILDER.priceId is hardcoded instead of reading from env, update it to use `process.env.NEXT_PUBLIC_STRIPE_BUILDER_PRICE_ID!`.

Step 7: Add the env var to Vercel:
```bash
npx vercel env add NEXT_PUBLIC_STRIPE_BUILDER_PRICE_ID production < <(echo "price_XXXXX")
```
Or if vercel CLI is not available, note this as a manual follow-up.

IMPORTANT: If NEXT_PUBLIC_STRIPE_BUILDER_PRICE_ID is already set and valid, simply verify it and move on. Do not create duplicates.
