<objective>
Inject recent chat history into the voice agent so FRED picks up where text left off when a user switches to voice.

Purpose: When a founder has been chatting with FRED and then starts a voice call, FRED should already know what they discussed in chat. Currently, the voice agent starts with zero context about prior chat conversations.

Output: A chat-voice bridge utility, updated call API that fetches and passes chat context, and updated voice agent that uses that context in its system prompt.
</objective>

  <name>Task 1: Create chat-voice bridge and update call API</name>
  <files>lib/fred/chat-voice-bridge.ts, app/api/fred/call/route.ts</files>
  <action>
Create `lib/fred/chat-voice-bridge.ts` with two exported functions:

1. `getChatContextForVoice(userId: string): Promise<{ preambleBlock: string; lastTopic: string | null }>`:
   - Use `createServiceClient()` from `@/lib/supabase/server`
   - Query `fred_episodic_memory` table for the user's last 10 conversation entries (event_type = 'conversation'), ordered by created_at desc
   - Filter to entries where content.channel is 'chat' or missing (default to chat)
   - Format them into a voice-friendly preamble block like:
     ```
     ## RECENT CHAT CONTEXT
     The founder has been chatting with you. Here's what was discussed recently:
     [User]: "..."
     [Fred]: "..."
     ...
     Reference these topics naturally. Do not repeat information verbatim — acknowledge you remember and build on it.
     ```
   - Truncate each message to 200 chars max to keep the voice prompt concise
   - Return empty preambleBlock string if no recent chat entries exist

2. `getLastDiscussedTopic(entries: ChannelConversationEntry[]): string | null`:
   - Takes the recent entries and extracts the last user message's first sentence or first 80 chars
   - Returns null if no user messages found
   - This is used for the "Last discussed: [topic]" UI indicator

Then update `app/api/fred/call/route.ts`:
   - Import `getChatContextForVoice` from `@/lib/fred/chat-voice-bridge`
   - After authentication and before room creation, call `getChatContextForVoice(userId)`
   - Pass the `preambleBlock` in the agent dispatch metadata: `{ userId, callType, chatContext: result.preambleBlock }`
   - Include `lastDiscussedTopic: result.lastTopic` in the JSON response alongside existing fields (token, url, room, etc.)
   - This is a non-blocking addition — if getChatContextForVoice fails, log warning and proceed with empty context

Do NOT modify the existing voice agent in this task — that is Task 2.
