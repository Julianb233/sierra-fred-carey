<task type="auto">
  <name>Task 1: Quick Reality Lens Wizard Page</name>
  <files>
    app/dashboard/reality-lens/quick/page.tsx
  </files>
  <action>
Create `app/dashboard/reality-lens/quick/page.tsx` as a "use client" component.

**Component Structure:**

1. **State Management:**
   - `step`: number (0 = intro, 1-6 = questions, 7 = analyzing, 8 = results)
   - `answers`: object matching QuickAnswers shape (initially empty strings)
   - `result`: QuickAssessmentResult | null
   - `loading`: boolean
   - `error`: string | null
   - `redirecting`: boolean (for already-completed check)

2. **On Mount -- Completion Check:**
   - Fetch `GET /api/fred/reality-lens/quick`
   - If response `data.complete === true`, redirect to `/dashboard` using `router.replace`
   - Show a toast via sonner: "You've already completed your reality check"

3. **Detect First-Time Mode:**
   - Read `searchParams` for `first=true` query param
   - If `first=true`, show a brief intro screen (step 0) with:
     - Heading: "Quick Reality Check"
     - Subtext: "Before we start your journey, let's understand where you are. This takes about 2 minutes."
     - "Let's Go" button to advance to step 1
   - If not first-time (no query param), start at step 1 directly

4. **Question Flow (steps 1-6):**
   - Progress bar at top: `Progress` component showing `(step / 6) * 100`
   - "Question {step} of 6" label
   - One question per screen using data from `QUICK_QUESTIONS` (imported from `@/lib/fred/reality-lens-quick`)
   - For text questions: `Textarea` with placeholder, min 10 chars to enable Next
   - For select questions: `Select` component with options from QUICK_QUESTIONS
   - "Back" button (ghost variant, disabled on step 1) and "Next" button (Sahara orange)
   - On step 6, "Next" becomes "Analyze My Idea" with RocketIcon
   - Use Framer Motion `AnimatePresence` with slide transitions between steps (slideLeft on next, slideRight on back)

5. **Analyzing State (step 7):**
   - Full-screen centered loading with Sahara gradient background
   - Cycling messages every 2s: "Evaluating your idea...", "Analyzing market fit...", "Checking readiness...", "Identifying gaps..."
   - Use `useEffect` with `setInterval` to cycle messages
   - Submit answers via `POST /api/fred/reality-lens/quick` with `{ answers }` body
   - On success: set result, advance to step 8
   - On error: set error, go back to step 6

6. **Results Screen (step 8):**
   - **Score Display:** Large score number with color coding (green >= 75, amber >= 50, red < 50), matching existing Reality Lens `getScoreColor` pattern
   - **Stage Badge:** "Your starting point: {stage name}" with desert-themed badge (use stage colors from `stage-config.ts`)
   - **Verdict Label:** Display `result.verdictLabel` as subheading
   - **Gaps Card:** Orange-tinted card with header "What you need to figure out before investors will listen:" and bulleted gap list. Use ExclamationTriangleIcon. This is the "spook them" moment per Fred's strategy.
   - **Strengths Card:** Green-tinted card with header "What's working for you:" and bulleted strengths list. Use CheckCircledIcon.
   - **Next Action:** Highlighted call-to-action card with `result.nextAction`
   - **Action Buttons:**
     - Primary: "Continue to Your Journey" -> navigates to `/dashboard` (Sahara orange button)
     - Secondary: "Get Detailed Analysis" -> link to `/dashboard/reality-lens` (ghost button)

7. **Styling:**
   - Use shadcn/ui: Card, Button, Badge, Progress, Select, SelectContent, SelectItem, SelectTrigger, SelectValue, Textarea
   - Tailwind classes throughout
   - Sahara brand color #ff6a1a for primary actions
   - Framer Motion for step transitions
   - Responsive: single column, works well on mobile (important for event QR code flow)
   - Max width: `max-w-2xl mx-auto` to keep focused
