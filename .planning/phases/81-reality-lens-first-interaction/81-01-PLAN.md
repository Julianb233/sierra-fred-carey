1. Create `lib/fred/reality-lens-quick.ts`:

   - Define and export `QuickAnswers` type:
     ```ts
     interface QuickAnswers {
       idea: string           // Q1: "Describe your startup idea in 2-3 sentences"
       targetCustomer: string // Q2: "Who is your target customer?"
       revenueModel: 'subscription' | 'marketplace' | 'services' | 'ads' | 'other'  // Q3
       customerValidation: 'none' | 'informal' | 'interviews-10plus' | 'paying-customers'  // Q4
       prototypeStage: 'idea-only' | 'mockups' | 'mvp' | 'launched'  // Q5
       biggestChallenge: string  // Q6
     }
     ```

   - Export `QUICK_QUESTIONS` array with 6 question objects, each having: `id`, `question`, `type` ('text' | 'select'), and `options` (for select type). The questions are:
     1. "Describe your startup idea in 2-3 sentences" (text)
     2. "Who is your target customer?" (text)
     3. "How will you make money?" (select: subscription/marketplace/services/ads/other)
     4. "Have you talked to potential customers?" (select: none/informal conversations/10+ structured interviews/paying customers already)
     5. "Do you have a working prototype?" (select: idea only/mockups or wireframes/working MVP/launched product)
     6. "What is your biggest challenge right now?" (text)

   - Export `QuickAssessmentResult` type:
     ```ts
     interface QuickAssessmentResult {
       overallScore: number  // 0-100
       stage: OasesStage
       gaps: string[]        // 3-5 items: "Here's what you need to figure out"
       strengths: string[]   // 2-3 items: "What's working for you"
       nextAction: string    // Single recommended next step
       verdictLabel: string  // e.g. "Early Stage - Focus on Clarity"
     }
     ```

   - Export `mapScoreToStage(score: number, customerValidation: string, prototypeStage: string): OasesStage`:
     - score < 30 OR (customerValidation === 'none' AND prototypeStage === 'idea-only') -> "clarity"
     - score 30-59 OR customerValidation is 'informal' -> "validation"
     - score 60-79 AND prototypeStage is 'mvp' or 'launched' -> "build"
     - score >= 80 AND customerValidation is 'paying-customers' -> "launch"
     - Default fallback: "clarity"
     - Never return "grow" (users earn that by completing all stages)

   - Export `quickAssessIdea(answers: QuickAnswers): Promise<QuickAssessmentResult>`:
     - Use Vercel AI SDK `generateObject` with `@ai-sdk/anthropic` (follow pattern from existing `lib/fred/reality-lens.ts`)
     - Use a concise system prompt: "You are a startup advisor. Evaluate this founder's readiness based on their answers. Score 0-100 based on idea clarity, customer validation, revenue model viability, and execution readiness. Identify gaps that would prevent investor interest. Be encouraging but honest."
     - Pass the 6 answers as user message content
     - Define zod schema for structured output matching QuickAssessmentResult (without stage -- compute stage separately)
     - After LLM returns score, call `mapScoreToStage(score, answers.customerValidation, answers.prototypeStage)` to set stage
     - Keep LLM call fast -- use `maxTokens: 500` and temperature 0.3

     - score 50, paying-customers, idea-only -> "validation" (score range wins)
   - Do NOT test `quickAssessIdea` (requires LLM -- test in integration)
