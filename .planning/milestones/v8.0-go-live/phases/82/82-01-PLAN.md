This means the voice agent doesn't know the founder's name, company, stage, market, challenges, etc.

    1. In `lib/fred/chat-voice-bridge.ts`:
       - Import `loadFounderMemory` from `@/lib/fred/active-memory` and `formatMemoryBlock` from `@/lib/fred/active-memory` (or the appropriate formatter)
       - In `getChatContextForVoice()`, after loading chat context, also load the founder's structured memory via `loadFounderMemory(userId)`
       - Add a new field `founderContext: string` to the `ChatVoiceContext` interface
       - Format the founder memory into a concise block (use `formatMemoryBlock` if available, otherwise build a compact string from the 14 fields)
       - Return `founderContext` alongside `preambleBlock` and `lastTopic`

    2. In `lib/voice/chat-context-loader.ts`:
       - In `formatChatForPreamble()`, accept an optional `founderContext?: string` parameter
       - If `founderContext` is provided and non-empty, prepend it before the "Recent chat context" section:
         ```
         About this founder:
         {founderContext}

         Recent chat context (the user chatted with you in text before this call):
         ...
         ```
       - This ensures the voice agent knows WHO the founder is, not just what they chatted about

    3. In `app/api/voice/context/route.ts`:
       - The GET handler already calls `loadChatContextForVoice` and `formatChatForPreamble`
       - Update it to also pass the founder context through (from the bridge or by calling loadFounderMemory directly)
       - Add `founderContext` to the JSON response

    4. In `workers/voice-agent/agent.ts`:
       - The agent already fetches context via `fetchChatContext()` and injects it via `buildFredVoicePromptWithContext()`
       - Verify this chain works with the enriched preamble. The existing code appends `chatContext` to the prompt. Since we're enriching the preamble string itself (in formatChatForPreamble), no changes needed in the agent unless the response shape changed.
       - If `founderContext` is now a separate field in the API response, update `fetchChatContext()` to handle it and merge it with the preamble.

    IMPORTANT: The voice agent runs in a separate Node.js worker process (not Next.js). It calls the /api/voice/context endpoint over HTTP. So all founder memory enrichment should flow through the API response, not through direct module imports in the worker.
