  <action>
    Two changes to make intro preparation more useful and FRED more proactive:

    1. **Create AI-personalized intro prep API** (`app/api/boardy/intro-prep/route.ts`):
       - GET endpoint accepting `?matchId=xxx` query parameter
       - Auth: require authenticated user via `createClient()` from `@/lib/supabase/server`
       - Fetch the match record from `boardy_matches` table to get matchName, matchType, matchDescription, metadata
       - Fetch the user's profile (company name, stage, sector) from `profiles` table
       - Use `generateText` from `ai` package (Vercel AI SDK) with the cheapest model (`getModelForTier` with Free tier to get mini model) to generate personalized intro content:
         - System prompt: "Generate a personalized introduction preparation guide for a startup founder meeting an investor/advisor."
         - User prompt includes: founder's company name, stage, sector + match name, type, description, focus area
         - Output: JSON with `{ callScript: string, emailTemplate: string, talkingPoints: string[] }`
       - Return `{ success: true, prep: { callScript, emailTemplate, talkingPoints } }`
       - If match not found or not owned by user, return 404
       - Rate limit: use Upstash ratelimit if available, otherwise skip

    2. **Enhance IntroPrepCard to use AI-personalized content** (`components/boardy/intro-prep-card.tsx`):
       - Add a "Personalize with AI" button that fetches from `/api/boardy/intro-prep?matchId=xxx`
       - When clicked, show loading state, fetch personalized content, and replace the template content with AI-generated content
       - Store the personalized content in component state so it persists during the session
       - Keep the existing template content as the default (shown before AI personalization)
       - Add a small badge "AI Personalized" when personalized content is active
       - The existing template-based content from `generateCallScript`/`generateEmailTemplate` remains the default fallback

    3. **Enhance FRED proactivity** (`lib/ai/prompt-layers.ts`):
       - In the BOARDY MATCH AWARENESS section, add a proactive rule: "When the founder has completed their venture journey (100% completion) and has active matches, proactively mention in the FIRST message of a new conversation: 'By the way, you have [N] investor/advisor matches ready for introductions. Want me to help you prepare?' -- but only once per conversation, not repeatedly."
       - Add a rule: "When the founder asks about next steps, fundraising timeline, or what to do now, always reference their Boardy matches as a concrete next action."

    WHY AI-personalized prep: The static templates are useful but generic. An AI endpoint that knows the founder's company AND the specific match can generate much more actionable prep (e.g., "Based on your SaaS in the healthcare space, lead with your HIPAA compliance story when talking to Sarah Chen at MedTech Ventures").
  </action>
