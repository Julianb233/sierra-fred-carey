<task type="auto">
  <name>Task 3: Create unified stage-gate prompt builder with proactive guidance</name>
  <files>lib/oases/stage-gate-prompt.ts</files>
  <action>
Create a NEW file `lib/oases/stage-gate-prompt.ts` that builds the complete stage-aware prompt block for FRED's system prompt. This is the single source of truth for stage-gate prompt injection in v8.0 — it replaces the separate prompt blocks from `lib/ai/stage-gate/redirect-templates.ts` and the existing `buildStageGatePromptBlock` in `stage-validator.ts`.

The file exports two functions:

**1. `buildStageAwarePromptBlock(currentStage: OasesStage): string`**

Builds a comprehensive prompt block that tells FRED:
- Current stage name, tagline, and position (e.g., "Stage 2 of 5: Validation — Test your assumptions")
- Current stage's step agenda (from STAGE_CONFIG) as a numbered list with descriptions
- Allowed topics (current + earlier stages)
- Restricted topics (later stages) with examples
- Proactive guidance instructions:
  - "You are in PROCESS-DRIVEN MENTOR mode. Do NOT wait for the founder to ask questions."
  - "Your default behavior: TELL the founder what to focus on, offer specific next actions from the current stage agenda."
  - "Open conversations with: 'Based on where you are in your journey, here's what matters most right now...'"
  - "Drive toward completing the current stage's steps. Ask the step's priority questions one at a time."
  - "When they complete something, acknowledge it and move to the next action."
- General conversation allowance (greetings, wellbeing, mindset always allowed)

**2. `buildStageRedirectBlock(validation: StageValidationResult): string`**

Takes the result from `validateStageAccess` and builds the redirect or override prompt block:
- When `allowed: false`: Builds a STAGE-GATE REDIRECT block with the redirect message, detected stage, current stage, and guidance for current stage steps
- When `isOverride: true`: Builds a STAGE-GATE MENTOR OVERRIDE block instructing FRED to help but be transparent about gaps
- When `allowed: true` and not override: Returns empty string (no redirect needed)

Import types from `@/types/oases` and `./stage-config`. Import `StageValidationResult` from `./stage-validator`.

Do NOT import from `lib/ai/stage-gate/` — this is the replacement, not a wrapper.
