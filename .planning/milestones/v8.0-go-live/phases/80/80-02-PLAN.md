   ```
   const stageValidation = validateStageAccess(message, currentOasesStage, stageRedirectCounts)
   ```

4. **Build redirect block using new builder**: Replace the inline `stageRedirectBlock` construction (~lines 734-746) with:
   ```
   const stageRedirectBlock = buildStageRedirectBlock(stageValidation)
   ```

5. **Persist redirect counts**: When a redirect occurs (stageValidation.allowed === false), increment the redirect count for that topic in conversation state. Use fire-and-forget pattern:
   ```
   if (!stageValidation.allowed && stageValidation.redirectKey) {
     const updatedCounts = { ...stageRedirectCounts }
     updatedCounts[stageValidation.redirectKey] = (updatedCounts[stageValidation.redirectKey] || 0) + 1
     // Fire-and-forget: persist updated counts to conversation state
     updateStageRedirectCounts(userId, updatedCounts).catch(err =>
       console.warn("[FRED Chat] Failed to persist stage redirect counts:", err)
     )
   }
   ```

6. **Add stageAwareBlock to fullContext array**: Insert `stageAwareBlock` in the fullContext assembly array, right after `stageRedirectBlock` (if present) and before `founderContext`. This ensures FRED always has stage context:
   ```
   let fullContext = [
     stageRedirectBlock,   // Phase 80: redirect (highest priority when active)
     stageAwareBlock,      // Phase 80: always-on stage awareness + proactive guidance
     founderContext,
     stepGuidanceBlock,
     // ... rest unchanged
   ].filter(Boolean).join("\n\n");
   ```

7. **Update context truncation priority**: In the truncation block (~lines 773-784), add `stageAwareBlock` at priority 2 (after stageRedirectBlock and founderContext). The stage-aware block should survive truncation because it's critical for process-driven behavior.

8. **Add updateStageRedirectCounts helper**: If `lib/db/conversation-state.ts` doesn't have a function to update stage redirect counts, add a simple helper that updates the `mode_context` JSONB column on `fred_conversation_state`. Use upsert pattern:
   ```
   export async function updateStageRedirectCounts(userId: string, counts: Record<string, number>): Promise<void> {
     const supabase = createServiceClient()
     // Read-modify-write the mode_context to merge stageRedirectCounts
     const { data } = await supabase
       .from("fred_conversation_state")
       .select("mode_context")
       .eq("user_id", userId)
       .single()
     const existing = (data?.mode_context as Record<string, unknown>) || {}
     await supabase
       .from("fred_conversation_state")
       .update({ mode_context: { ...existing, stageRedirectCounts: counts } })
       .eq("user_id", userId)
   }
   ```

   If `fred_conversation_state` table doesn't exist for this user, skip silently (the conversation state is created lazily).
