<task type="auto">
  <name>Task 1: Create admin sentiment data layer and API endpoint</name>
  <files>
    lib/db/sentiment-admin.ts
    app/api/admin/sentiment/route.ts
  </files>
  <action>
    Create the data layer and API for the admin sentiment dashboard.

    1. **Create `lib/db/sentiment-admin.ts`** with three query functions using `createServiceClient()` (service role bypasses RLS):

       a. `getSentimentOverview(days: number = 7)`: Returns aggregated stats:
          - Total signals count
          - Count by label (positive, neutral, negative, frustrated)
          - Count of interventions triggered
          - List of "high stress founders" — users with avg stress_level > 0.5 in the period. For each: user_id, signal_count, avg_stress_level, last_signal_at. Join profiles table to get founder name (display_name or email). Limit to top 20.

       b. `getInterventionLog(limit: number = 50)`: Returns recent intervention events:
          - Query `sentiment_signals` WHERE `intervention_triggered = true`, ordered by `created_at DESC`
          - For each: id, user_id, label, stress_level, topics, created_at
          - Join profiles to get founder name

       c. `getUserSentimentHistory(userId: string, days: number = 30)`: Returns per-user time series:
          - All signals for the user in the period, ordered by `created_at ASC`
          - Fields: label, confidence, stress_level, topics, intervention_triggered, created_at

    2. **Create `app/api/admin/sentiment/route.ts`** — GET endpoint:
       - Import `requireAuth` and verify the user is admin (check existing admin auth pattern in `app/api/admin/audit-log/route.ts`)
       - Accept query params: `view` (overview | interventions | user-history), `days` (number), `userId` (string, for user-history)
       - Call the corresponding function from `lib/db/sentiment-admin.ts`
       - Return JSON response
       - Handle errors gracefully with 500 responses

    Follow existing patterns from `lib/db/feedback-admin.ts` and `app/api/admin/audit-log/route.ts` for auth and query structure.
  </action>
  <verify>
    - `npm run build` compiles without errors
    - `grep -n "getSentimentOverview\|getInterventionLog\|getUserSentimentHistory" lib/db/sentiment-admin.ts` shows all 3 functions
    - `grep -n "requireAuth\|admin" app/api/admin/sentiment/route.ts` shows admin auth check is present
  </verify>
  <done>
    Admin API at /api/admin/sentiment returns sentiment overview (aggregated stats + high-stress founders), intervention log, and per-user history. Service client bypasses RLS. Admin auth required.
  </done>
