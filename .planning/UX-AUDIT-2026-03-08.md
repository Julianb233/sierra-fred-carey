## What's Working Well

- **Homepage hero is impactful** — "What if you could create a unicorn, all by yourself?" with Sahara orange accent is compelling and visually strong
- **Pricing page is excellent** — 3-tier layout is clear, copy differentiates tiers on outcomes not features ("time-to-clarity"), $0/$99/$249 structure is easy to understand
- **Get Started wizard is smooth** — 3-step flow (stage → challenge → signup) reduces friction, progress dots in top-right, back button works, password requirements shown inline
- **Dashboard is feature-rich** — FRED chat with sentiment badges, feedback thumbs, quick-start chips, voice/call options, "Today's Focus" with stage-aware guidance, progress tracking with 3 metric cards
- **Oases journey visualization** — 5-stage path (Clarity → Grow) is well-designed, makes the product feel like a guided experience rather than a chatbot
- **Backend API validation is solid** — Event registration properly returns 400/404 for bad input, admin routes properly return 401

## Fix Before Ship — P1 High

### P1-1: Homepage shows 5-8 seconds of black screen on load
**Page:** https://joinsahara.com
**Issue:** Initial page load shows a near-black screen for 5-8 seconds before content appears. Likely a heavy intro animation. First-time visitors will bounce.
**Fix:** Either remove/shorten the intro animation to <1.5s, or add a preloader skeleton showing brand elements immediately.

### P1-2: PWA install banner overlaps navigation on event page
**Page:** https://joinsahara.com/event/palo-alto-2026
**Issue:** "Install Sahara" toast overlaps the "See it in Action" nav link. On the standalone event page, the main site navbar shouldn't appear at all (per plan spec, layout should be minimal/standalone).
**Fix:** Event layout should hide the main navbar. If keeping it, suppress PWA prompt on event pages.

## Fix This Week — P2 Medium

### P2-1: Dashboard greeting shows email prefix instead of full name
**Page:** /dashboard
**Issue:** "Hey ux-audit-20260308" — should show the user's display name from `user_metadata.full_name`, not the email prefix.
**Fix:** Check `user.user_metadata?.full_name || user.email?.split('@')[0]` — the name was set during account creation but not displayed.

### P2-2: /event/nonexistent returns 200 instead of 404
**Page:** /event/nonexistent
**Issue:** Navigating to a non-existent event slug returns HTTP 200. Should return 404 with a "not found" page.
**Fix:** Ensure `notFound()` in the page component triggers a proper 404 HTTP status.

### P2-3: /dashboard accessible without auth (returns 200)
**Page:** /dashboard (unauthenticated)
**Issue:** Hitting `/dashboard` without auth returns 200 instead of redirecting to `/login`. The page likely renders a loading state then client-side redirects, but the server response should be 302.
**Fix:** Add middleware redirect for unauthenticated access to `/dashboard/*` routes.

### P2-4: Event landing page shows full site nav instead of standalone layout
**Page:** /event/palo-alto-2026
**Issue:** The event page plan specified a minimal layout with no dashboard nav. Currently shows the full site navbar (Pricing, Features, About, Login, Get Started).
**Fix:** The `app/event/[slug]/layout.tsx` should not include the main navbar component.

### P2-5: Fred Cary quote on event page uses initials avatar instead of headshot
**Page:** /event/palo-alto-2026
**Issue:** "FC" circle avatar appears instead of Fred Cary's actual headshot photo. The About page has the real photo — reuse it here.
**Fix:** Import the Fred Cary headshot image and use it in the event landing quote section.

## Backlog — P3 Polish

### P3-1: Dark/light mode toggle icon inconsistency
**Page:** All pages
**Issue:** The theme toggle icon (sun/moon) in the navbar could be more clearly labeled. Minor accessibility gap.

### P3-2: "See it in Action" nav link styling
**Page:** All pages
**Issue:** This link is styled in orange (like a CTA) but it's a nav link, not the primary action. "Get Started Free" is the primary CTA. Having two visually prominent items in the nav creates competition.

### P3-3: Sidebar scroll indicator
**Page:** /dashboard
**Issue:** The sidebar shows "Coaching" near the bottom with a tiny scroll indicator. Consider reducing sidebar items or making scroll more obvious.
