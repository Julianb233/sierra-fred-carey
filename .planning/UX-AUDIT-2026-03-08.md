## P0 — Fix NOW

### P0-1: Homepage body blank on initial load
- **Page:** https://joinsahara.com
- **Behavior:** Page loads with header/nav visible but body content is blank for ~15 seconds. Content only appears after scroll or interaction.
- **Impact:** First-time visitors see an empty page and bounce immediately.
- **Likely cause:** Heavy GSAP/Framer Motion animations blocking render, or SSR hydration mismatch with `"use client"` + `dynamic()` on `app/page.tsx`.

## P1 — Fix before anyone sees

### P1-1: Dead "Join the Waitlist" CTA
- **Page:** Homepage hero section
- **Behavior:** Primary CTA button says "Join the Waitlist" but Sahara is live — clicking it goes to a waitlist form instead of signup/get-started.
- **Impact:** Confuses visitors, undermines credibility, loses signups.
- **Fix:** Change CTA to "Get Started Free" → link to `/get-started`.

### P1-2: Stats contradiction — $50M vs $100M
- **Page:** Homepage social proof section
- **Behavior:** One section says "$50M+ in funding secured" and another says "$100M+ in deals facilitated." Contradictory or unclear distinction.
- **Impact:** Erodes trust. Investors and founders notice inconsistencies.
- **Fix:** Clarify or unify the metrics with proper context labels.

### P1-3: Tier naming inconsistency
- **Page:** Pricing page vs dashboard
- **Behavior:** Pricing page shows "Starter / Growth / Scale" tiers but dashboard and internal code use "Free / Pro / Studio."
- **Impact:** Users can't map what they're paying for to what they see in-app.
- **Fix:** Unify tier naming across all surfaces.

### P1-4: Features nav chevron misleading
- **Page:** Global navigation
- **Behavior:** "Features" nav item has a dropdown chevron but clicking it navigates to a page instead of opening a dropdown menu.
- **Impact:** UX anti-pattern — chevrons indicate expandable menus, not links.
- **Fix:** Either add a dropdown or remove the chevron.

## P2 — Fix this sprint

### P2-1: Auth guards don't redirect to intended URL
- **Page:** All protected routes (`/dashboard`, `/chat`, etc.)
- **Behavior:** After login, users land on `/dashboard` instead of the page they originally tried to access.
- **Impact:** Friction for returning users clicking deep links from emails/notifications.
- **Fix:** Pass `redirectTo` query param through the auth flow.

### P2-2: Bare-bones 404 page
- **Page:** Any invalid URL (e.g., `/nonexistent`)
- **Behavior:** Shows a plain, unstyled 404 page with no navigation or branding.
- **Impact:** Dead-end experience. Users can't recover.
- **Fix:** Add branded 404 with search, nav links, and "Go to Dashboard" CTA.

### P2-3: Legal pages say "Fred Carey" not "Sahara"
- **Page:** `/terms`, `/privacy`
- **Behavior:** Legal copy references "Fred Carey" as the entity instead of "Sahara" or the proper legal entity name.
- **Impact:** Looks unprofessional, may have legal implications.
- **Fix:** Update legal entity references to match business registration.

### P2-4: Homepage too long — no section anchors
- **Page:** Homepage
- **Behavior:** Very long page with 8+ sections but no anchor links or in-page navigation.
- **Impact:** Users can't jump to relevant content; high scroll fatigue.
- **Fix:** Add anchor-based section nav or a sticky table of contents.

### P2-5: Several API routes return 404 in production
- **Routes:** `/api/health`, various internal endpoints
- **Behavior:** Routes that work locally return 404 on production deploy.
- **Impact:** Health checks and monitoring integrations fail silently.
- **Fix:** Verify Vercel deployment includes all API routes; check for build-time exclusions.
